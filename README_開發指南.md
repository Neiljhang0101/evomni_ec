## 🗺️ 模組開發依賴順序

> 這不是建議，**是強制的開發前置條件**。上層未完成不得進入下層開發。

```
Layer 0：Evomni 主程式（共用模組）
    會員認證 / 媒體庫 / 發信系統 / 文章管理 / 數據中心基礎建設
         ↓
Layer 1：全域設定與商店基礎建設（Part 1）
         ↓
    ┌────────────┬────────────┬────────────┐
    ↓            ↓            ↓
Layer 2a       Layer 2b     Layer 2c
產品中心        會員管理      金物流串接設定
(Part 2)       (Part 6)     (串接設定後台)
    └────────────┴────────────┴────────────┘
         ↓（三者都完成才能進入）
Layer 3：訂單管理（Part 3）
         ↓
    ┌─────────────┐
    ↓             ↓
Layer 4a         Layer 4b
行銷活動          數據中心
(Part 4)         (Part 5)
```

**關鍵依賴說明：**
- 金物流串接設定必須在訂單管理前完成，否則訂單無法走完支付流程
- 會員管理（Part 6）是行銷自動化的前置條件，Part 6 未完成，Part 4 進階功能無法驗收
- 數據中心（Part 5）依賴訂單、產品、會員三個模組的資料寫入

## 🖥️ 如何使用 Prototype

Prototype 為單一 `.html` 文件，不需要安裝任何東西：

1. 用瀏覽器（Chrome / Edge / Safari）直接開啟 `.html` 檔案
2. 左側導覽切換各子頁面
3. 所有互動（篩選、表單填寫、Drawer 開關）都已實作

## ⚙️ 技術規格約定

**技術棧：**
- 前端：Vue 3 + Vite + Element Plus + Tailwind CSS
- 後端：Laravel 10 / PHP 8.3
- 資料庫：MySQL 5.x+
- 快取：Redis（用於 Queue Job、分散式鎖）

**PRD 中的技術規格原則：**
> DB Schema、API 路由為**建議參考**，工程師可依技術判斷調整。重大結構變更請回寫對應 PRD 的 §8.5，並通知 PM 更新。

**幾個全系統共用的技術決策（已定案，不可更動）：**
- 庫存扣減使用 `SELECT ... FOR UPDATE` 悲觀鎖（訂單並發安全）
- 軟刪除統一使用 `deleted_at` timestamp，不做物理刪除
- 所有第三方 API Token 使用 AES-256 加密儲存，禁止明文存 DB
- 多規格產品的 `products` 和 `product_skus` 必須在同一 Transaction 內寫入
- 自動上下架為每分鐘 Scheduled Job（Laravel Schedule），不使用 MySQL Event Scheduler
