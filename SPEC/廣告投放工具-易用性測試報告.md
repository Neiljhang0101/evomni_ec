## 三、問題清單（依嚴重程度排序）

---

### 🔴 Critical

**問題 C-01：Info Banner 導向路徑錯誤**

- **影響：** T1、所有任務（每個 Tab 都顯示此 Banner）
- **位置：** `html/廣告投放工具.html` 第 793 行
- **問題描述：** Banner 文字寫「廣告追蹤碼設定請前往『全域設定 > 行銷追蹤碼管理』」，連結文字為「前往行銷追蹤碼管理 →」。UX Spec 及 PRD 明確要求路徑為「行銷管理 > 行銷工具設定」，且這是 Hard Constraint。商家若真的想去設定追蹤碼，會被引導到錯誤的頁面。
- **根本原因：** 開發期間 Spec 定義的頁面路徑（行銷管理 > 行銷工具設定）與 Prototype 實作時使用的導覽路徑（全域設定 > 行銷追蹤碼管理）不一致，未同步確認。
- **建議修正：** 將第 793 行的文字改為「廣告追蹤碼（Facebook Pixel / GA4 / LINE Tag）設定請前往『行銷管理 > 行銷工具設定』，不在此頁設定。」；連結文字改為「前往行銷工具設定 →」。同時確認 Sidebar 中「行銷工具設定」是否存在對應頁面。

---

**問題 C-02：Feed Token 管理功能完全缺失**

- **影響：** T4（Feed 設定流程）
- **位置：** `html/廣告投放工具.html` 第 496–565 行（FeedCard 元件）
- **問題描述：** UX Spec Screen 3 規定每張 Feed 卡片需要「Feed URL Token 管理」摺疊區塊，包含說明文字（「如果 Feed URL 外洩，可重新生成 Token 讓舊 URL 立即失效。」）、「重新生成 Feed Token」danger 按鈕，以及點擊後開啟確認 Dialog。目前 Prototype 兩張卡片都沒有此區塊。這是安全性功能，PRD §8.2 也特別標注。
- **根本原因：** FeedCard 元件未實作 Token 管理區塊，`showTokenDialog` 狀態有定義（第 475 行）但從未使用。
- **建議修正：** 在兩張 Feed 卡片底部（說明連結下方）加上摺疊 Token 管理區塊；加上確認 Dialog，Dialog 需要包含 PRD 要求的警告文字（「⚠️ 此操作無法復原」）。

### 🟠 Major

**問題 M-01：必填欄位驗證錯誤未顯示，僅靠按鈕 disabled 引導**

- **影響：** T2（UTM 連結生成）
- **位置：** `html/廣告投放工具.html` 第 202、220–224、380–384 行
- **問題描述：** `errors` state 在第 202 行初始化，但整個元件中從未被 `setErrors()` 更新。當三個必填欄位（utm_source、utm_medium、utm_campaign）未填時，「複製完整連結」和「複製短網址」按鈕只會顯示 `opacity: 0.5`，無法點擊。但用戶不知道缺少的是哪個欄位。UX Spec 要求：點擊複製但欄位未填時，顯示欄位紅框加錯誤訊息（如「請填寫廣告來源（utm_source）」）。
- **根本原因：** 驗證觸發邏輯未實作；`handleCopyfull` 直接 return 不顯示任何錯誤。
- **建議修正：** 在 `handleCopyfull` 中加入驗證邏輯，對空白必填欄位呼叫 `setErrors({ src: '請填寫廣告來源（utm_source）', ... })`；讓欄位 border 在 error 狀態顯示紅色（現有 `borderColor: errors.src ? '#F56C6C' : '#DCDFE6'` 已備妥，只差觸發）。或至少在按鈕下方加一行 helper text 說明「請先填寫必填欄位（來源、媒介、廣告活動）」。

---

**問題 M-02：utm_source 與 utm_medium 缺少自動補全建議**

- **影響：** T2（UTM 連結生成）
- **位置：** `html/廣告投放工具.html` 第 326–333 行
- **問題描述：** UX Spec 規定 utm_source 使用 `el-autocomplete` 並提供 7 個預設建議（facebook、google、line、instagram、email、sms、qrcode）；utm_medium 提供 8 個建議。目前兩個欄位都是普通 `<input>`，非專業用戶不知道要輸入什麼格式。「快速模板」可部分緩解問題，但直接自行填寫時沒有任何提示。
- **根本原因：** Prototype 簡化了元件實作，未加入 datalist 或 dropdown 建議機制。
- **建議修正：** 加入 HTML5 `<datalist>` 或自訂 dropdown 建議列表，提供預設選項；在欄位 placeholder 加一句「點擊可選擇常用平台」讓用戶知道有建議可用。

---

**問題 M-03：Feed 首次使用空狀態缺失，始終顯示硬編碼更新時間**

- **影響：** T4（Feed 設定）
- **位置：** `html/廣告投放工具.html` 第 548 行
- **問題描述：** Feed 卡片底部的「上次更新」區塊始終顯示硬編碼的「2026/05/14 10:00　產品數量：245 件」。UX Spec 規定首次使用時（Feed 從未生成）應顯示「尚未生成，請點擊『立即重新生成 Feed』」，引導用戶知道要先執行生成。新開通的商家首次看到頁面時會以為已有 Feed 在運作，但實際上可能什麼都沒設定。
- **根本原因：** 未實作 `isFirstTime` 狀態，所有數據均為靜態 mock。
- **建議修正：** 加入切換控制（可以是 toggle button 或 checkbox 模擬「首次使用」狀態），讓測試者能看到空狀態的呈現。

---

**問題 M-04：廣告來源分析表格無排序 UI，非專業用戶難以快速找出最佳廣告活動**

- **影響：** T5（廣告效益判斷）
- **位置：** `html/廣告投放工具.html` 第 670–740 行
- **問題描述：** UX Spec 規定訂單數欄位預設降序排列（最多訂單在上），且多個欄位可點擊排序。目前 Prototype 的表格標頭沒有排序箭頭或點擊互動，用戶只能靠目視比較各行數字。雖然預設資料順序是 facebook 在上（92 筆），但若未來資料順序變動，用戶完全無法重排。
- **根本原因：** `th` 樣式物件（第 620 行）未加入 `cursor: pointer`，也沒有排序狀態管理。
- **建議修正：** 在「訂單數」欄位標頭加排序箭頭圖示（降序為預設，用深色表示已選中），至少讓 Prototype 展示出排序 UI 的樣貌。

### 🟡 Minor

**問題 N-01：QR Code 預覽尺寸為 96px，規格要求 128px**

- **影響：** T2（QR Code 展示）
- **位置：** `html/廣告投放工具.html` 第 406–407 行
- **問題描述：** `<QRPlaceholder size={96} />` 與 UX Spec「128×128px」不符，視覺上偏小，且右側說明文字（「填寫必填欄位後顯示 QR Code」）在現有 96px 尺寸下顯得比例不協調。
- **建議修正：** 改為 `size={128}`。

**問題 N-02：utm_term 欄位 placeholder 縮寫格式不符 CLAUDE.md 規則**

- **影響：** T2
- **位置：** `html/廣告投放工具.html` 第 350 行
- **問題描述：** 目前 placeholder 為「選填，通常用於關鍵字廣告（搜尋引擎行銷）的關鍵字」，將縮寫（SEM）展開為中文但未保留縮寫。CLAUDE.md 規定「以白話中文標示，並在括號內附上原始縮寫」，正確格式應為「關鍵字廣告（SEM）」，而非「搜尋引擎行銷」。UX Spec 原文也寫 `（SEM）`。
- **建議修正：** 改為「選填，通常用於關鍵字廣告（SEM）的關鍵字」。

---

**問題 N-03：產品搜尋 placeholder 縮寫格式錯誤**

- **影響：** T2（產品選擇）
- **位置：** `html/廣告投放工具.html` 第 303 行
- **問題描述：** 目前 placeholder 為「搜尋產品名稱或產品編號（貨號）…」，UX Spec 原文為「搜尋產品名稱或 SKU…」。依 CLAUDE.md 規則，應寫成「產品名稱或產品庫存單位（SKU）」。目前「貨號」為貨品編號的口語說法，容易讓商家誤解。
- **建議修正：** 改為「搜尋產品名稱或產品庫存單位（SKU）…」。

---

**問題 N-04：Facebook Feed 說明連結文字「設定產品 Feed」應為「設定產品目錄」**

- **影響：** T4（Feed 設定學習）
- **位置：** `html/廣告投放工具.html` 第 560 行
- **問題描述：** Facebook 卡片說明連結為「如何在 Facebook Business Manager 設定產品 Feed 查看說明 ↗」，UX Spec 要求為「如何在 Facebook Business Manager 設定產品目錄」。Facebook 廣告平台官方名稱為「產品目錄（Catalog）」，使用「Feed」作為名稱與平台用語不一致，會讓用戶在廣告平台操作時找不到對應功能。另外說明文字和連結 CTA 被合併成一行，建議拆開（說明文字 + 獨立的「查看說明 ↗」連結）。
- **建議修正：** 改為「如何在 Facebook Business Manager 設定產品目錄」，並將「查看說明 ↗」獨立為連結文字。

---

**問題 N-05：儲存模板 Dialog 缺少字數顯示（show-word-limit）**

- **影響：** T3（模板儲存）
- **位置：** `html/廣告投放工具.html` 第 450 行
- **問題描述：** UX Spec 規定模板名稱欄位需顯示字數（maxlength=50，show-word-limit）。目前只有基本 input，用戶輸入時不知道還有多少字可用。
- **建議修正：** 在輸入框右下角加上字數顯示（如「0 / 50」）。

---

**問題 N-06：「直接流量 / 無追蹤標籤」說明文字為靜態文字而非 Hover Tooltip**

- **影響：** T5（廣告來源分析）
- **位置：** `html/廣告投放工具.html` 第 728–729 行
- **問題描述：** 說明文字「(未加追蹤標籤的連結)」以靜態小字方式顯示在欄位標題旁，而非 Hover Tooltip。UX Spec 規定使用 `el-tooltip`（Hover 觸發）。靜態文字佔用表格列空間，視覺上也較雜亂。
- **建議修正：** 改為 Hover 時顯示的 Tooltip，在「直接流量 / 無追蹤標籤」文字旁加一個 info 圖示，hover 後顯示說明。
