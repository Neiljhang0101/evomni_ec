### 🔴 Critical — 功能完全缺失，任務受阻

---

**C-01 「最高折扣上限」欄位未實作**

- **影響任務：** T1
- **位置：** `html/行銷中心.html`，`CouponFormPage` `tab === 'basic'`（約 Line 343–387）
- **問題描述：** 商家選擇「百分比折扣」後，UX Spec 要求動態顯示「最高折扣上限」欄位（NumberInput，帶 Tooltip）。目前的程式碼中，form state 無 `maxDiscount` 屬性，JSX 亦無任何條件式顯示此欄位的邏輯。商家無論如何操作，都無法輸入折扣上限。
- **根本原因：** 此功能在最新 commit（9297e3f）中未加入，CouponFormPage 基本設定 Tab 未包含此欄位。
- **建議修正：** 在 form state 加入 `maxDiscount: ''`；在 `tab === 'basic'` 的 JSX 中，於折扣數值欄位後方加入條件顯示邏輯：`{form.discountType === 'percent' && <FormField label="最高折扣上限"...>}`。

---

**C-02 「排除產品」欄位未實作**

- **影響任務：** T2
- **位置：** `html/行銷中心.html`，`CouponFormPage` `tab === 'condition'`（約 Line 389–416）
- **問題描述：** UX Spec 的使用條件 Tab 要求新增「排除產品」多選搜尋欄位，讓商家指定哪些產品不適用此券。目前的程式碼中，整個 condition tab 僅有：最低訂單金額、最低購買數量、適用產品範圍（單選下拉）、不可疊加 Switch，完全沒有「排除產品」相關欄位或 state。
- **根本原因：** 未實作。
- **建議修正：** 在 form state 加入 `excludeProducts: []`；在 `tab === 'condition'` 加入帶遠端搜尋的多選 Select（with Tooltip），並以灰色 Tag 顯示已選排除產品。

**M-02 「生日贈券」觸發選項缺少進階電商包鎖定標示**

**M-03 「可與其他優惠疊加」Switch 語義與規格相反**

- **影響任務：** T2
- **位置：** `html/行銷中心.html`，Line 408–413
- **問題描述：** Prototype 的欄位標籤為「不可與其他優惠同時使用」（負向表述），Switch ON = 「已開啟 — 不可疊加使用」，OFF = 「已關閉 — 可與其他優惠疊加」。UX Spec 的欄位標籤為「可與其他優惠疊加」（正向表述），Switch ON = 「已開啟 — 可與促銷活動同時使用」，OFF = 「已關閉 — 不可與其他優惠同時使用」。兩者的 Switch ON/OFF 含義完全相反，視覺上同樣顯示為「開啟（綠色）」但行為語義相反，容易導致商家誤設。
- **根本原因：** 原始 Prototype 採用負向欄位名稱（exclusive），與本次補齊 UX Spec 的正向表述不一致。
- **建議修正：** 將欄位標籤改為「可與其他優惠疊加」，並調整 Switch 開關狀態說明文字至與 UX Spec 一致（ON = 可疊加，OFF = 不可疊加）。
