## Part 1 — Design Brief

### Product Vision

購物車與結帳流程是 Evomni 電商前台消費者旅程的核心轉換段落——從「決定要買」到「訂單成立」的完整體驗。本流程包含三個前台頁面：購物車頁（確認品項與計算金額）、結帳頁（填寫收件資訊、選擇付款方式、套用優惠）、訂單確認頁（付款或訂單成立確認）。設計目標是讓消費者在最短的步驟內安心完成購買，同時讓訪客與會員都有流暢體驗，不因身份差異而遇到阻礙。

### Target User

**Primary（已登入會員）：** 完成選購的消費者，已有帳號，期望能快速帶入地址、套用優惠碼或點數，確認後送出。

**Secondary（訪客，未登入）：** 從廣告或分享連結直接進入商品頁、加入購物車，不想建立帳號但仍要完成購買；部分情境（如 0 元商品、點數折抵）需引導登入。

### Design Goals

| # | Goal | How to test |
|---|---|---|
| 1 | 消費者從購物車頁進入結帳、完成送出的全流程不超過 5 分鐘 | 計時觀察從點擊「前往結帳」到訂單確認頁出現 |
| 2 | 消費者能在購物車頁看到「為什麼不能結帳」的原因，並知道如何解除阻擋 | 觀察有下架商品時，消費者是否能自行找到移除方法 |
| 3 | 消費者在結帳頁填完表單前，能隨時在側欄看到「目前應付金額」 | 觀察消費者是否有反覆滾動確認金額的行為 |
| 4 | 訪客完成結帳後，能從確認信或確認頁找到查詢訂單的方式 | 詢問訪客如何查詢訂單狀態 |
| 5 | 優惠碼套用失敗時，消費者能理解失敗原因並知道下一步 | 觀察消費者面對錯誤提示時的反應 |

### Design Principles

1. **減少離開路徑** — 結帳頁不提供「繼續購物」以外的離開連結；購物車頁的修改動作（刪除）透過 EvomniAlert 確認，不打斷主流程
2. **即時金額可見** — 訂單摘要側欄在桌機端 sticky 常駐；手機端折疊至頁尾但在送出前展開確認
3. **訪客不強制登入** — 除 0 元商品外，訪客不需登入即可完成結帳；登入引導以提示橫幅呈現，非遮罩阻擋
4. **錯誤要說清楚** — 所有錯誤提示寫具體原因而非「系統錯誤」；優惠碼失敗提示寫出失敗原因分類

### Design System Reference

- **設計系統：** Evomni 前台設計系統（`frontend-design-system/`）
- **字體：** Inter（拉丁文字）+ Noto Sans TC（中文），基礎字級 16px，最小字級 14px
- **圖示：** Material Icons Outlined（CDN）
- **色彩：** CSS 自訂屬性，引用 `colors_and_type.css`
- **彈窗 / 通知：** EvomniAlert（`evomnialert/`）— 所有確認對話、操作回饋均使用此元件，禁止自製 modal 或 toast
- **頁面背景：** `var(--fill-300)`（米白）
- **頁面最大寬度：** `var(--container)`

### Accessibility Requirements

- **Target WCAG level:** AA
- **Known constraints:**
  - 所有文字最小 14px，不得使用更小字級
  - 表單欄位 focus 狀態需清楚顯示（`border-color: var(--ink-900)`）
  - 數量 + / - 按鈕最小 32×32px，手機觸控需留足夠間距
  - 下架商品的灰色蒙層需附文字說明（不能只靠顏色傳達狀態）
  - 錯誤訊息需有顏色以外的視覺區別（icon 或前綴文字）

### Hard Constraints

- ⚠️ 禁止使用 Emoji 字元（所有文案、標籤、說明文字）
- ⚠️ 前台頁面不使用後台設計系統（Element Plus、`components/` 資料夾元件）
- ⚠️ 所有彈窗、確認對話、通知訊息一律使用 EvomniAlert，禁止自製
- ⚠️ 英文縮寫需附中文說明（如：「成交金額（GMV）」），但 CTA 按鈕英文全大寫（如 CHECKOUT）例外
- ⚠️ 結帳頁不顯示優惠計算 11 步驟的技術細節，只顯示最終折扣金額

### Out of Scope

- 優惠計算引擎 11 步驟的前台 UI（僅顯示結果）
- 超商取貨地圖選門市 UI（由金流商 SDK 提供）
- 外部付款頁（金流商頁面，非 Evomni 設計範圍）
- 電子發票 API 串接細節
- 訂單建立後的管理操作（屬後台訂單管理範圍）
- 一頁式商店的結帳 UI 佈局（邏輯共用，視覺由 LP 範本決定）

### Screen 1：購物車頁

**Purpose：** 消費者確認購物車品項、調整數量或移除品項、查看優惠預估與配送選項，並前往結帳。
**Entry points：** Header 的購物車 icon、商品頁加入購物車後的引導連結
**Primary user goal：** 確認品項正確後，點擊「前往結帳」

#### Information Hierarchy

```
H1（最顯眼）：SHOPPING BAG / 購物車（N 件）
H2（次要）：品項列表（商品名稱 + 規格 + 小計）
Primary CTA：前往結帳
Secondary CTA：繼續購物、查看活動（未達優惠門檻時）
Supporting info：商品小計、運費、優惠折扣、訂單小計（側欄）
```

#### Actual Copy

**Page / Section Headings**
- 頁面眉標（Eyebrow）：`SHOPPING BAG`（英文大寫，寬字距）
- 頁面主標：`購物車`（後接商品數量：`購物車（3 件）`）
- 欄位表頭：`品項` / `數量` / `小計`（英文大寫：`ITEM` / `QUANTITY` / `SUBTOTAL`）

**Button Labels**
- Primary CTA：`前往結帳`
- 繼續購物：`繼續購物`
- 加購品加入：`加入購物車`
- 點數兌換：`兌換`
- 優惠碼套用：`套用`
- 刪除按鈕：垃圾桶 icon（無文字，靠 EvomniAlert 提供確認文字）
- 優惠碼移除：icon × 按鈕（aria-label：`移除優惠碼`）
- 已下架商品移除：`移除`

**EvomniAlert — 刪除確認（`EvomniAlert.show()`）**
- type：`warning`
- title：`確定要移除「{商品名稱}」？`（動態帶入商品名）
- desc：`此操作無法復原`
- onConfirm：執行移除品項
- 按鈕文字：`OK`（元件預設，不可修改）

**商品狀態標籤**
- 已下架（圖片遮罩）：`已下架`（全大寫，白字）
- 已下架（說明文字）：`此商品已下架，請移除後繼續結帳`（灰色文字，圖片旁）
- 庫存警示：`僅剩 {N} 件`（黃色文字，`var(--warning)`）
- 庫存不足自動調整：`庫存不足，已自動調整為 {N} 件`（黃色文字）

**配送地區選項**（RegionSelector）
- 台灣本島
- 離島（金門、馬祖、澎湖）
- 選中樣式：黑底白字；切換後同步過濾 DeliverySelector 可選選項及費率

**配送方式選項**（DeliverySelector，依選取地區過濾）
- 宅配到府（2–4 個工作天）— 台灣本島 NT$80 / 離島 NT$200；台灣本島滿 NT$1,500 免運
- 7-ELEVEN 超商取貨（取貨門市自行選擇）— NT$65，全地區
- 全家超商取貨（取貨門市自行選擇）— NT$65，全地區
- 冷凍宅配（低溫配送，2–5 天）— NT$200，僅台灣本島
- 已達免運門檻時顯示：`免運費`（`var(--success)` 綠色）

**付款方式選項**（PaymentSelector，依配送方式過濾）
- 信用卡（一次付清）— 適用所有配送方式
- LINE Pay — 適用所有配送方式
- ATM 轉帳（7 天內完成轉帳）— 適用所有配送方式
- 超商代碼繳費（72 小時內繳款）— 不適用冷凍宅配
- 超商取貨付款 — 僅超商取貨方式
- 貨到付款 — 僅宅配到府

**訂單摘要側欄**
- 欄位標籤：`商品小計` / `優惠折扣` / `點數折抵` / `運費`
- 未選配送方式時：`待選擇`
- 達免運門檻：`免運費`
- 折扣金額前綴：`-`（如 `- NT$ 200`）
- 小計區段標籤：`訂單小計`
- 優惠碼輸入標籤：`優惠代碼`（英文大寫：`PROMO CODE`）
- 優惠碼 placeholder：`請輸入優惠代碼`
- 點數折抵標籤：`紅利點數折抵`（英文大寫：`POINTS`）
- 可用點數說明：`可用 {N} 點`
- 點數換算說明：`1 點 = NT$ 1，最多折抵 NT$ {N}`
- 點數輸入 placeholder：`輸入折抵點數`

**免運費進度提示**（FreeShippingHint）
- 文字：`再消費 NT$ {N} 即享免運費`（`{N}` 粗體顯示）
- 進度百分比：`{N}%`

**優惠活動區**（PromotionsSection）

已達成優惠（顯示於上方，綠色背景）：
- 標籤：`已達成`（綠色 badge）
- 說明：`折扣已自動套用至訂單`
- 已省金額：`NT$ {N}`（右側，`var(--success)` 色）

未達成優惠（顯示於下方）：
- 區段標題：`未符合優惠活動`
- 差額提示：`再消費 NT$ {N} 即可達標 — {活動說明}`
- CTA：`查看活動`

**加購品 / 點數兌換區**（AddOnsSection）
- Tab 標籤 1：`加購品`
- Tab 標籤 2：`紅利兌換品（會員專屬）`
- 點數兌換品顯示：`{N} 點`

**防呆提示**（Disabled 狀態）
- 有下架商品時：按鈕 Disabled，按鈕文字改為 `請移除已下架商品`
- 空購物車時：按鈕 Disabled

**Empty State（購物車為空）**
- 眉標：`YOUR BAG IS EMPTY`
- Headline：`您的購物車是空的`
- Subtext：`去挑選喜歡的商品吧！`
- CTA：`繼續購物`

**Form Fields（優惠碼 / 點數）**

| 欄位 | Placeholder | Helper | Error |
|------|-------------|--------|-------|
| 優惠代碼 | `請輸入優惠代碼` | （無） | `優惠代碼無效，請確認後重新輸入` |
| 紅利點數折抵 | `輸入折抵點數` | `1 點 = NT$ 1，最多折抵 NT$ {N}` | （數量超過上限時自動截至上限） |

**Error States（優惠碼）**

| 系統狀態 | 顯示文字 |
|---------|---------|
| 代碼不存在 | `優惠代碼無效，請確認後重新輸入` |
| 未達消費門檻 | `此優惠需消費滿 NT$ {X} 才可使用` |
| 已使用過 | `此優惠代碼已使用過` |
| 超過使用期限 | `此優惠代碼已過期` |
| 不適用此商品 | `購物車中無符合此優惠的商品` |

**Loading State**
- 頁面載入：skeleton（品項卡片佔位），無文字

#### Component List

| 元件名稱 | 規格 / 變體 | 互動行為 |
|---------|-----------|---------|
| CartItem 品項卡片 | 正常 / 庫存警示 / 已下架 / 已下架+移除按鈕 | 數量調整即時更新側欄金額；刪除 icon 觸發 EvomniAlert 確認 |
| QtyControl 數量控制 | 有效 / disabled（下架商品）| 點擊 + / - 更新數量；不可低於 1，不可超過 min(99, 庫存) |

| DeliverySelector 配送方式 | Radio card 列表（依地區篩選）| 點選後側欄即時更新運費；免運達標時顯示綠色「免運費」|

#### Screen States

| 狀態 | 觸發條件 | 呈現方式 |
|------|---------|---------|
| 正常（含商品） | 購物車有有效商品 | 品項列表 + 配送地區選擇 + 配送方式 + 付款方式 + 訂單摘要側欄；CTA 可點擊 |
| 庫存警示 | 有商品庫存 ≤ 5 件 | 品項下方顯示黃色「僅剩 N 件」|
| 自動調整數量 | 購物車品項數量 > 剩餘庫存 | 黃色文字「庫存不足，已自動調整為 N 件」|
| 含已下架商品 | 購物車有商品已下架 | 該品項灰色蒙層 + 「商品已下架」+ 「移除」按鈕；「前往結帳」Disabled |
| 空購物車 | 購物車無任何品項 | Empty State：icon + 說明文字 + 「繼續購物」CTA |
| 優惠碼套用成功 | 輸入有效代碼後點「套用」| 顯示折扣碼名稱 + 折扣金額；右側「× 移除」|
| 優惠碼套用失敗 | 輸入無效 / 不符合條件代碼 | 紅色 error 文字（依原因顯示對應訊息） |
| 刪除確認中 | 點擊刪除 icon | EvomniAlert warning dialog 出現 |
| 頁面載入中 | 首次載入 / 重新驗證庫存 | skeleton placeholder |

#### Interaction Annotations

| User action | Result |
|-------------|--------|
| 點擊刪除 icon | 觸發 `EvomniAlert.show({ type: 'warning', title: '確定要移除...', onConfirm: onDelete })`；按 OK 後品項從列表消失，側欄金額更新 |
| 調整數量 | 即時更新該品項小計與側欄訂單摘要金額 |
| 切換配送地區（台灣本島 / 離島）| DeliverySelector 過濾可用選項，若原選配送方式不支援新地區則自動重選第一個可用項 |
| 選擇配送方式 | 側欄運費即時更新；若達免運門檻顯示「免運費」；PaymentSelector 同步過濾不可用付款方式 |
| 選擇付款方式 | 記錄選擇，送出前與配送方式一同傳入結帳頁 |
| 輸入優惠碼後按「套用」或 Enter | 驗證代碼；成功顯示折扣說明；失敗顯示紅色原因文字 |
| 點擊「× 移除」（優惠碼）| 清除折扣碼，側欄金額更新 |
| 輸入點數折抵數量 | 即時計算折抵金額，側欄更新；超過可用點數自動截至上限 |
| 點擊「前往結帳」（有效狀態）| 跳轉至結帳頁 |
| 點擊「前往結帳」（Disabled）| 無反應；hover 顯示 tooltip 說明原因 |
| 點擊「繼續購物」| 返回商品列表頁 |

### Screen 2：結帳頁

**Purpose：** 消費者填寫收件資訊、選擇發票類型與付款方式，確認金額後送出訂單。配送地區、配送方式、優惠碼、點數折抵均在購物車頁完成選擇。
**Entry points：** 購物車頁點擊「前往結帳」
**Primary user goal：** 完整填寫必要資料，確認金額後送出訂單

#### Information Hierarchy

```
H1（最顯眼）：CHECKOUT / 結帳
Primary CTA：確認送出
Secondary CTA：修改購物車
Supporting info：訂單摘要側欄（即時金額、商品清單）
區塊標題：收件資訊 / 發票設定 / 付款方式
```

#### Actual Copy

**Page / Section Headings**
- 頁面眉標：`CHECKOUT`
- 頁面主標：`結帳`
- 區塊標題 1：`收件資訊`（Eyebrow 小標：`DELIVERY INFO`）
- 區塊標題 2：`發票設定`（Eyebrow 小標：`INVOICE`）
- 區塊標題 3：`付款方式`（Eyebrow 小標：`PAYMENT`）
- 側欄標題：`訂單摘要`

**Button Labels**
- Primary CTA：`確認送出`
- Loading 狀態：`處理中…`（按鈕 Disabled + spinner）

### Screen 1 — 購物車頁

```
請幫我設計「購物車頁」的 UI。這是一個前台消費者端的桌面寬版頁面（1280px），採左右兩欄佈局：左欄（約 2/3 寬）顯示購物車品項列表，右欄（約 1/3 寬）為 sticky 訂單摘要側欄（含優惠碼輸入、點數折抵、金額明細）。

最重要的元素是：(1) 品項卡片列表，每張卡片含商品圖（80×80px）、名稱規格、數量增減控制、小計、刪除 icon；(2) 訂單摘要側欄的「前往結帳」Primary CTA 按鈕。

頁面背景為米白（var(--fill-300)），字體 Inter + Noto Sans TC，字級最小 14px，無圓角設計語言。設計上方顯示免運費進度條橫幅（例如「再消費 NT$ 500 即享免運費」），品項下方有加購品橫向捲動區。

請參考上傳文件中 Screen 1「購物車頁」的完整規格，特別注意品項的三種狀態：正常、含庫存警示（黃色文字）、含已下架商品（灰色蒙層）。
```

### Screen 2 — 結帳頁

```
請幫我設計「結帳頁」的 UI。這是一個前台消費者端的桌面寬版頁面（1280px），採左右兩欄佈局：左欄（2/3 寬）為由上往下的表單分區，右欄（1/3 寬）為 sticky 訂單摘要側欄（含購物車品項清單及金額明細）。

左欄由上往下共 3 個區塊，每個區塊有獨立標題（Eyebrow 小標 + 區塊名稱）：
① 收件資訊（姓名、手機、Email、縣市/鄉鎮聯動 Select、詳細地址、備註）
② 發票設定（4 個 Radio 選項：雲端載具、自然人憑證、公司用途、捐贈）
③ 付款方式（Radio Card：信用卡（一次付清）、LINE Pay、ATM 虛擬帳號、超商代碼繳費、貨到付款）

頁面頂端有一條淺藍色訪客提示橫幅：「已有帳號？登入以享有點數回饋與地址快速填入」附「登入」按鈕。

配送地區、配送方式、優惠碼、點數折抵已在購物車頁完成，右欄側欄直接顯示已套用的優惠及最終應付金額。

最重要的元素是頁面最底部的「確認送出」Primary 大按鈕。字體 Inter + Noto Sans TC，米白背景，所有欄位 focus 時邊框變深色（var(--ink-900)），無圓角。請參考文件 Screen 2「結帳頁」的完整規格。
```

### Screen 3 — 訂單確認頁

```
請幫我設計「訂單確認頁」的 UI。這是消費者完成結帳付款後看到的確認頁，屬於前台消費者端，置中單欄佈局（非左右雙欄）。

頁面主要內容區由上往下：
1. 大型成功勾選 icon（綠色，var(--success)）
2. 眉標「ORDER CONFIRMED」
3. 主標題「訂單已成立！」（粗體大字）

整體風格簡潔、成功感強、無干擾元素。字體 Inter + Noto Sans TC，米白背景，無圓角。請參考文件 Screen 3「訂單確認頁」的規格。
```
