## Part 1 — Design Brief

### Product Vision

前台商品頁是消費者與商品第一次「正式接觸」的場域，涵蓋商品列表（探索、篩選、比較）與商品詳情（深度了解、決策、加購）兩個核心頁面。設計目標是讓消費者以最低的認知負擔找到想要的商品，並在詳情頁獲得足夠的資訊與信任感，順暢進入購物車流程。列表頁側重「高效瀏覽」，詳情頁側重「信任建立與轉換」。

### Target User

**主要用戶：** 25–45 歲女性消費者，習慣網購護膚品，在乎成分與評論，會在多頁面比較後才決策；對頁面響應速度和視覺整潔度有較高期待。

**次要用戶：** 男性消費者（送禮情境），依賴評論和品牌說明作為決策依據，較少主動篩選分類。

### Design Goals

| # | 目標 | 驗證方式 |
|---|------|---------|
| 1 | 消費者從列表頁找到目標商品並進入詳情頁的時間不超過 30 秒 | 計時觀察從進入列表頁到點擊商品卡片 |
| 2 | 消費者在詳情頁能不需要滾動即看到商品名稱、價格與主要 CTA | 視覺熱圖、first viewport 分析 |
| 3 | 規格選擇防呆引導有效：消費者在嘗試加入購物車後能知道「哪個規格未選」 | 觀察消費者是否能在 3 秒內看到防呆提示並完成修正 |
| 4 | 手機端消費者不需要捲到頁面中段即可完成加入購物車 | 觀察手機用戶是否使用底部 sticky CTA bar |
| 5 | 評論區的星等分佈圖讓消費者能快速判斷評論可信度 | 詢問消費者如何解讀評論區信息 |

### Design Principles

1. **內容優先，裝飾從簡** — 商品圖與名稱是最重要的視覺元素，不以過多裝飾干擾判斷
2. **狀態清楚標示** — SOLD OUT、庫存不足、特價等狀態以視覺差異（顏色、標籤、遮罩）明確傳達，不只依賴顏色
3. **手機操作第一** — 底部 sticky CTA、橫向分類 chips、縮圖可點等互動以手指操作為設計基準
4. **信任積累** — 評論區、成分說明、信任標章置於轉換漏斗後段，強化決策信心

### Design System Reference

- **設計系統：** Evomni 前台設計系統（`frontend-design-system/`）
- **字體：** Inter（拉丁）+ Noto Sans TC（中文），基礎字級 16px，最小字級 14px
- **圖示：** Material Icons Outlined（CDN）
- **色彩：** CSS 自訂屬性，引用 `colors_and_type.css`
- **彈窗 / 通知：** EvomniAlert（`evomnialert/`）
- **頁面背景：** `var(--fill-300)`（米白）
- **英文 UI 標籤：** 全大寫 + 寬字距（SHOP ALL、SOLD OUT、SKINCARE）

### Accessibility Requirements

- **WCAG Level:** AA
- 所有文字最小 14px
- 圖片需提供 alt 文字
- 售完遮罩需同時顯示 SOLD OUT 文字（不能只靠半透明遮罩表達）
- 規格按鈕 disabled 狀態需有文字刪除線 + 降低對比度，不能只靠顏色
- 評分星等需提供文字描述（如「4.8 分，共 124 則評論」）
- 手機觸控按鈕最小 36×36px

### Hard Constraints

- 禁止使用 Emoji
- UI 文案不得出現縮寫（SKU、UV、PV），需以白話中文標示
- 不自製 modal 或 toast，一律使用 EvomniAlert
- 不顯示庫存確切數字（僅顯示「僅剩 N 件」的低庫存警示）

### Out of Scope（本版）

- 商品搜尋結果頁（獨立頁面）
- 商品評論撰寫功能（Phase 2）
- 心願清單頁面（僅有加入動作，清單頁 Phase 2）
- 一頁式商店（`/lp/:slug`，獨立 PRD）

### Screen 1：商品列表頁（Product Listing Page）

**目的：** 讓消費者從全站商品中以分類、搜尋、價格快速縮小選擇範圍，找到目標商品並點入詳情。

**進入方式：**
- Header 導覽列點擊「SHOP」、「保養」、「彩妝」、「香氛」
- 商品詳情頁點擊麵包屑分類連結
- 首頁「立即選購」CTA

**使用者主要目標：** 找到想購買的商品類型，或在特定分類中選出適合商品。

---

#### 1.6 互動標註

- **商品卡片點擊：** 任意位置點擊（含圖片、名稱）皆跳轉 `/products/:slug`（prototype：`商品詳情.html`）
- **收藏 icon：** stopPropagation 阻止冒泡；已加入時顯示實心 favorite icon；觸發 EvomniAlert success toast
- **分類切換：** 立即過濾，不需確認；H1 與麵包屑同步更新
- **搜尋框：** onChange 即時過濾，不需按 Enter；清空搜尋框恢復全部
- **排序切換：** 立即重排，不需確認；不影響篩選條件
- **手機 chips 選中清除：** count bar 出現「清除」文字連結，點擊重置為「全部商品」

### Screen 2：商品詳情頁（Product Detail Page）

**目的：** 提供消費者完整商品資訊（圖片、說明、成分、評論），引導選擇規格並加入購物車或直接購買。

**進入方式：**
- 商品列表頁點擊商品卡片
- 購物車頁點擊商品縮圖或商品名稱
- 相關商品（Related Products）點擊

**使用者主要目標：** 了解商品、確認規格、加入購物車或立即購買。

#### 2.5 互動標註

**圖片區**
- 縮圖點擊：切換主圖，同時重置放大鏡 zoomPos
- 主圖 mousemove（桌面）：計算游標在圖片上的相對位置（x%、y%），驅動放大窗口 backgroundPosition
- 放大窗口：position absolute，left: calc(100% + 16px)，top: 0，280% 背景尺寸
- 手機：不顯示放大窗口（.pdp-zoom-panel { display: none }）

**規格選擇**
- 每次點擊規格值：清除該規格群的 specError
- 加入購物車 / 立即購買點擊時：遍歷所有 SPEC_GROUPS，有未選的設 specError；任一錯誤時中止
- 規格錯誤顯示：error_outline icon + 錯誤文字（紅色）

**手機 sticky CTA bar**
- 頁面底部固定，zIndex 50
- 左側：售價（大）+ 庫存狀態文字（小，灰色）
- 右側：加入購物車 + 立即購買（或售完時：到貨通知我）
- 主體 main 加 padding-bottom: 100px 避免遮蓋底部評論

**加購滾動**
- 使用 overflowX: auto 容器
- 滑鼠橫滑或觸控滑動
- 每張加購卡片 flexShrink: 0, width: 160px
