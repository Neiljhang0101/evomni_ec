## 1. 功能概覽

選單管理讓設計師／網站管理員建立並維護前台導覽選單的樹狀結構。

核心行為：
- 建立多種類型的選單節點（文章列表、產品列表、自訂連結…等 8 種）
- 以拖曳方式調整節點順序與層級
- 每個語系（繁中 / EN）維護獨立選單結構
- 每個節點可獨立設定標題、自訂網址、可見度、開啟方式、SEO

## 2. 節點資料結構

### 2.1 MenuNode（前端資料格式）

```ts
interface MenuNode {
  id:       string;          // 唯一識別碼
  name:     string;          // 前台顯示名稱
  type:     MenuNodeType;    // 節點類型（見 2.2）
  url?:     string;          // 僅 type='link' 使用；其他類型由系統自動產生
  visible:  boolean;         // 前台可見度（false = 選單隱藏，但直連網址仍可訪問）
  openMode: 'same' | 'new'; // 換頁 | 另開新視窗
  children: MenuNode[];      // 子節點陣列（允許遞迴巢狀）
  // 設定面板儲存的額外欄位
  desc?:     string;         // 列表簡介
  seoTitle?: string;         // SEO 標題
  seoDec?:   string;         // SEO 描述
  // 來源參考 ID（後端用於關聯實際內容）
  sourceId?: string;         // 對應的文章分類 ID / 產品分類 ID / 表單 ID / 系統頁面 ID
}
```

### 2.2 MenuNodeType（節點類型定義）

| type | 中文名稱 | 說明 | 是否可有子節點 |
|---|---|---|---|
| `group` | 群組 | 純導覽群組，無對應內容頁 | 是 |
| `link` | 自訂連結 | 指向外部或自訂網址 | 否 |
| `article-list` | 文章列表 | 對應文章分類（顯示該分類下所有文章列表） | 是 |
| `article-page` | 文章單頁 | 對應單篇文章內容頁 | 否 |
| `product-list` | 產品列表 | 對應產品分類（顯示該分類下所有產品列表） | 是 |
| `product-page` | 產品單頁 | 對應單一產品內容頁 | 否 |
| `form` | 表單 | 對應指定表單頁面 | 否 |
| `system-page` | 系統單頁 | 系統固定頁面（隱私權政策、服務條款等） | 否 |

### 2.3 MenuStore（後端儲存結構）

```ts
interface MenuStore {
  menus: {
    [lang: string]: MenuNode[];  // lang: 'zh' | 'en' | 其他語系碼
  };
  langVisible: {
    [lang: string]: boolean;     // 各語系的前台可見度（全域開關）
  };
}
```

**說明：**
- `langVisible` 是語系層級的全域開關，與節點層級的 `visible` 獨立運作
- 語系可見度關閉時，整個語系的選單對前台不顯示，不論節點個別 `visible` 設定為何

## 4. 左側面板 — 選單樹

### 4.1 語系 Tab 列

```
[繁中] [EN]              顯示中 [Switch]  [複製 EN 架構] [新增項目]
```

- Tab 切換時清空 `selectedId`（右側設定面板回到空狀態）
- **可見度 Switch**：`langVisible[lang]` 的全域開關，切換前台整個語系選單的顯示狀態
- **複製架構按鈕**：複製相反語系的樹狀結構至當前語系，複製時遞迴將所有節點的 `visible` 設為 `false`
- **新增項目按鈕**：開啟「新增項目 Modal」（見第 6 節），`defaultParent = null`（加入第一層）

### 4.2 樹狀節點 Row

每個節點列高 48px，由左至右元件：

| 位置 | 元件 | 說明 |
|---|---|---|
| 1 | 拖曳把手（6 點）| cursor: grab；觸發拖曳（見第 7 節） |
| 2 | 展開／折疊箭頭 | 僅有子節點時顯示；點擊 toggle `expanded[id]` |
| 3 | 節點名稱（flex:1）| 點擊設定 `selectedId`，右側顯示對應設定面板 |
| 4 | 類型標籤 | 顏色依 type 決定（見 2.2 色彩定義） |
| 5 | 可見度 icon | 眼睛圖示，切換 `node.visible`；隱藏時顯示 warning 色 `#E6A23C` |
| 6 | 新增子項目（+）| 開啟新增 Modal，`defaultParent = node.id` |
| 7 | 刪除（垃圾桶）| 刪除節點及其所有子節點；如刪除的是 `selectedId` 則清空選擇 |

**節點視覺狀態：**

| 狀態 | 背景色 | 左側邊框 | 名稱顏色 |

- 樣式：虛線上邊框，藍色文字，hover 時淺藍背景

## 7. 拖曳排序

### 7.1 技術方式

使用 HTML5 Drag and Drop API（`draggable`、`onDragStart`、`onDragOver`、`onDrop`、`onDragEnd`）。

### 7.2 行為規格

| 事件 | 行為 |
|---|---|
| dragStart | 記錄 `draggingId`；`dataTransfer.effectAllowed = 'move'`；節點 opacity 降至 0.35 |
| dragOver | 計算滑鼠是否在目標節點上半段或下半段；更新 `dragOver: { id, pos: 'before' \| 'after' }` |
| dragLeave | 清空 `dragOver` |
| drop | 執行 `wmMove(tree, srcId, dstId, pos)`；清空拖曳狀態 |
| dragEnd | 清空 `draggingId` 和 `dragOver` |

### 7.3 視覺回饋

- 拖曳中的節點：`opacity: 0.35`
- Drop 目標 before 位置：目標節點**上方** 2px 藍線（`#409EFF`）
- Drop 目標 after 位置：目標節點**下方** 2px 藍線

### 7.4 移動邏輯

```
wmMove(tree, srcId, dstId, pos):
  1. 找出 src 節點（含其完整子樹）
  2. 從樹中刪除 src（遞迴刪除）
  3. 在 dstId 節點的前（before）或後（after）插入 src（遞迴插入）
```

**限制：**
- 不可拖曳到自身（`srcId === dstId` 時跳過）
- 跨語系不可拖曳（拖曳僅作用於當前語系的樹）

## 8. 匯入 / 匯出

### 8.1 匯出

- **觸發：** 右上角「匯出」按鈕
- **範圍：** 當前語系（`menus[lang]`）
- **格式：** JSON，美化排版（`JSON.stringify(data, null, 2)`）
- **檔名：** `menu-{lang}-{YYYY-MM-DD}.json`
- **機制：** Blob + `URL.createObjectURL` + 隱形 `<a>` 點擊下載 + `revokeObjectURL`

### 8.2 匯入

- **觸發：** 右上角「匯入」按鈕（hidden `<input type="file" accept=".json">`）
- **解析：** `FileReader.readAsText` + `JSON.parse`
- **驗證：** `Array.isArray(data)` 為 true，否則顯示 error Toast
- **行為：** 成功後以匯入資料**覆蓋**當前語系的選單（`menus[lang] = data`）
- **Toast：**
  - 成功：「選單已匯入」
  - 失敗：「檔案格式錯誤，請上傳正確的 JSON 檔案」

## 9. 語系複製規格

- **操作路徑：** 左側 Tab 列中「複製 {相反語系} 架構至此」按鈕
- **複製來源：** 相反語系的完整樹結構（深拷貝）
- **複製後：** 遞迴將所有節點的 `visible` 設為 `false`（預設隱藏，需手動逐節點開啟）
- **不複製：** `langVisible` 全域開關（各語系獨立維護）
- **Toast：** 「已複製 [語系名稱] 架構（預設全部隱藏）」

> **注意（工程師）：** 自訂網址（`url`）複製後是否需要依語系路徑自動調整（例如 `/en/about`）尚未定案，目前 Prototype 原樣複製，請與 PM 確認後再實作。

## 11. API 建議端點

> 以下為 Prototype 邏輯推算的建議端點，需後端確認後定案。

| 方法 | 路徑 | 說明 |
|---|---|---|
| GET | `/api/menus?lang={lang}` | 取得指定語系的完整選單樹 |
| PUT | `/api/menus?lang={lang}` | 儲存整棵選單樹（覆蓋）|
| PATCH | `/api/menus/{lang}/nodes/{id}` | 更新單一節點的設定欄位 |
| GET | `/api/menus/export?lang={lang}` | 匯出指定語系為 JSON |
| POST | `/api/menus/import?lang={lang}` | 匯入 JSON 覆蓋指定語系 |
| GET | `/api/article-categories` | 取得文章分類清單（新增 Modal 用）|
| GET | `/api/article-categories/{id}/pages` | 取得分類下的文章單頁清單 |
| GET | `/api/product-categories` | 取得產品分類清單 |
| GET | `/api/product-categories/{id}/pages` | 取得分類下的產品單頁清單 |
| GET | `/api/forms` | 取得表單清單 |
| GET | `/api/system-pages` | 取得系統單頁清單 |

**儲存策略：**
- 拖曳排序、新增節點、刪除節點等操作在前端即時更新 state，不即時送 API
- 統一由「儲存設定」按鈕或全頁「儲存選單」按鈕觸發一次性儲存
- 若要即時儲存，拖曳 `dragEnd` 後觸發 `PUT /api/menus`

## 14. 與原始 UX Spec 的差異整理

原始 Spec（`SPEC/選單架構優化-UX-Spec.md`）是針對**前台渲染邏輯**的設計文件，涵蓋左側選單樣式、上方標籤樣式等前台模板設計。本文件為**後台選單管理介面**的工程規格，兩者為不同範疇：

| 面向 | 原始 Spec | 本文件 |
|---|---|---|
| 對象 | 前台訪客 + 後台設計師（防呆警告） | 後台設計師 / 網站管理員 |
| 核心功能 | 前台選單渲染邏輯（左側選單 / 上方標籤 / 無樣式直通）| 後台選單節點的建立、編輯、排序、語系管理 |
| 節點類型 | 文章分類、單頁 | 8 種類型（見 2.2）|
| 設定面板 | 列表樣式選擇 + 孤兒節點警告 | 標題、網址、可見度、開啟方式、SEO |
| Prototype | 前台模板示意（3 種樣式 + 1 個後台元件）| 完整後台管理介面（約 700 行 JSX）|

> 前台渲染邏輯（樣式驅動渲染、首項直通、孤兒頁面防呆）仍為有效規格需求，但應另立工程文件說明前台實作，不在本文件範圍內。
