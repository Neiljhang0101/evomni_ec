## 三、問題清單（依嚴重程度排序）

### 🟠 Major

---

**M-01：刪除商品無確認彈窗**

- **影響：** T1、T2
- **位置：** html/購物車.html:148
- **問題描述：** CartItem 的垃圾桶按鈕直接呼叫 `handleDelete(item.id)`，立即從顯示清單移除商品，沒有確認步驟。
- **根本原因：** `onDelete` handler 只做 `setDeleted(prev => ({ ...prev, [key]: true }))` 而無確認 Dialog。
- **PRD 依據：** §6.1「刪除｜垃圾桶 icon，點擊後確認彈窗」
- **建議修正：** 點擊垃圾桶後顯示 inline confirm 或 Dialog：「確定要移除「{商品名}」嗎？」+ 確認 / 取消。

```jsx
// 修正示意（購物車.html L148 附近）
const [confirmDelete, setConfirmDelete] = useState(null);
// 點擊垃圾桶 → setConfirmDelete(item.id)
// Dialog 確認後 → handleDelete(confirmDelete)
```

---

**M-02：行動裝置（Mobile/Tablet）無可見的「確認送出」按鈕**

- **影響：** T4
- **位置：** html/結帳頁.html:602
- **問題描述：** Mobile 用途的送出按鈕被設定 `display: 'none'`；送出按鈕僅存在於右側 Sidebar（寬 320px）。Viewport 寬度小於約 720px 時，Sidebar 可能換行至底部或消失，導致無法找到送出入口。
- **根本原因：** `<div style={{ display: 'none' }}>` 明確隱藏了 Mobile 版送出按鈕，且未實作 RWD breakpoint 切換邏輯。
- **建議修正：** 改以 CSS media query 控制；桌機顯示 Sidebar 的送出，Mobile 顯示主欄底部的 sticky CTA bar。

```html
<!-- 修正：在 <style> 加入 -->
<style>
  @media (max-width: 767px) {
    .sidebar-submit { display: none; }
    .mobile-submit { display: block !important; }
  }
</style>
```

### 🟡 Minor

---

**N-03：FreeShippingHint 進度條違反 border-radius: 0 規則**

- **影響：** 設計合規
- **位置：** html/購物車.html:256–257
- **問題描述：** `borderRadius: 2` 違反全域 `border-radius: 0` 設計規範，設計師驗收時可能打回。
- **建議修正：** 移除 `borderRadius: 2`，改用方形進度條；或在 DS 例外清單中明文允許 progress bar 使用圓角。

---

**N-04：折扣碼失敗態固定顯示「代碼無效」，無法示範其他 4 種錯誤文案**

- **影響：** T3（折扣碼套用失敗狀態）
- **位置：** html/結帳頁.html:366–373
- **問題描述：** `FAILURE_MESSAGES` 物件（L332–338）已定義全部 5 種錯誤，但 `couponState === 'fail'` 時永遠取 `'invalid'` key，無法展示「未達門檻」、「已使用」、「已過期」、「不適用此商品」等文案。
- **PRD 依據：** §5.3.4 折扣碼失敗原因對應表（5 種）
- **建議修正：** StateBar 新增子狀態（如：折扣碼失敗-未達門檻 / 折扣碼失敗-已過期），或在 DiscountSection 內加一個下拉 Demo 選項。

---

**N-05：Normal 狀態無法展示「運費 NT$60」的情境**

- **影響：** 設計審查
- **位置：** html/購物車.html:264–277（ITEMS_BY_STATE mock data）
- **問題描述：** 全部 4 個 mock 狀態的 subtotal 均 ≥ 1000，因此「前往結帳」側欄的運費列永遠顯示「免運費」，審查者無法看到「NT$60」收費狀態。
- **建議修正：** 新增一個 mock 狀態（例：「單件商品」），subtotal = 590，顯示 NT$60 運費，同時觸發 FreeShippingHint。

---

**N-06：訂單確認頁按鈕文字與 Checklist 不一致**

- **影響：** 文案規範
- **位置：** html/訂單確認頁.html:235（訪客）、241（會員）
- **問題描述：** Checklist 要求「查看訂單」；Prototype 顯示「查詢訂單狀態」（訪客）與「查看我的訂單」（會員）。前者更清楚目的，但後者需與其他頁面統一。
- **建議修正：** 統一為：會員 → 「查看訂單詳情」；訪客 → 「查詢我的訂單」（與訪客查詢頁入口文案對齊）。

---

**N-07：商品圖片不可點擊，缺少連至商品詳情頁的跳轉**

- **影響：** T1（次要）
- **位置：** html/購物車.html:51（ImgPlaceholder component）
- **問題描述：** PRD §6.1 規定「商品圖 80×80px，點擊連至商品頁」，但 ImgPlaceholder 無 `<a>` 包覆或 onClick。
- **建議修正：** 以 `<a href="產品中心.html">` 包覆圖片（Prototype 階段連至商品詳情頁 stub）。

**E-08：0 元商品強制登入 Banner（CC-4）未在 Prototype 中示範**

- **位置：** html/結帳頁.html（缺少此狀態）
- **問題描述：** PRD §5.1.2 定案：訪客購物車含 0 元商品時，進入結帳頁顯示橫幅「免費商品需登入後才可完成結帳」。Prototype 的 StateBar 未包含此場景。
- **建議：** 新增「含免費商品（訪客）」狀態，展示此 Banner 及「登入」CTA。

---

**E-09：ATM / 超商代碼付款的逾時截止時間視覺強調不足**

- **位置：** html/結帳頁.html:63–65（PAYMENT_OPTIONS.note）
- **問題描述：** 付款截止時間以括號附在選項旁，對急迫性表達不足。使用者選擇後不清楚後果。
- **建議：** 選中 ATM / 超商代碼後，以橙色 Alert 提示「請於 N 小時內完成付款，逾期訂單將自動取消」。

---

**E-10：結帳側欄 Sticky 行為在長頁面效果未驗證**

- **位置：** html/結帳頁.html:477（`position: 'sticky', top: 72`）
- **問題描述：** 側欄設為 sticky top:72，但當商品欄位（發票、付款方式展開後）超過側欄高度時，側欄會在某個點停止 sticky 並跟著捲動，可能使「確認送出」按鈕不易找到。
- **建議：** 在長表單時，考慮讓整個 sidebar 以 `overflow-y: auto` + 固定高度 `calc(100vh - 72px)` 維持捲動獨立。
