### 🟠 Major

---

**問題 M-01：刪除按鈕 Disabled 時無 Tooltip，使用者不知道為什麼不能刪除**

- **影響任務：** T4
- **位置：** `html/行銷中心.html:1544`
- **問題描述：** 啟用中商店的「刪除」按鈕處於 disabled 狀態，但 hover 時沒有任何提示文字。一位不熟悉後台的使用者看到灰色按鈕，會不斷嘗試點擊，最終放棄並認為功能壞了。
- **根本原因：** `EvoBtn` 的 `disabled` prop 只讓按鈕變灰，沒有掛上 `title` 或 Tooltip 元件。
- **建議修正：** 在 `<EvoBtn>` 外層包一個 `<span title="請先停用此商店再刪除">` 以觸發原生 Tooltip；或在啟用狀態時顯示 disabled 狀態旁邊的說明小字。

```jsx
// 目前（html/行銷中心.html:1544）
<EvoBtn variant="secondary" size="sm" disabled={!canDelete} onClick={...}>刪除</EvoBtn>

// 建議
<span title={!canDelete ? '請先停用此商店再刪除' : ''}>
  <EvoBtn variant="secondary" size="sm" disabled={!canDelete} onClick={...}>刪除</EvoBtn>
</span>
```

**問題 M-02：Step 1「下一步：設計頁面」實際跳至 Step 4，中間略過兩個步驟**

- **影響任務：** T3
- **位置：** `html/行銷中心.html:1606`, `html/行銷中心.html:1778`
- **問題描述：** 按鈕文字寫「下一步：設計頁面」，使用者預期進入 GrapeJS 設計介面（Step 2），但實際上直接跳至「預覽與發布」（Step 4）。使用者看到 Stepper 顯示 Step 4 高亮，會以為自己做錯了事。
- **根本原因：** Step 2（GrapeJS）和 Step 3（加購設定）為 Stub，尚未建立。Prototype 直接接 Step 1 → Step 4。
- **建議修正：** 改用以下任一方案：
  - 按鈕文字改為「儲存並繼續（跳過 Steps 2–3）」，明確標注是 Prototype 捷徑
  - 或新增一個 Step 2 Stub 頁面，顯示「GrapeJS 頁面設計 — 規格待實作」並有「下一步」可繼續流程

### 🟡 Minor

**問題 Mi-01：麵包屑「行銷中心」連結至 LP 儀表板，語意錯誤**

- **影響任務：** T3、T5
- **位置：** `html/行銷中心.html:1626`, `html/行銷中心.html:1904`
- **問題描述：** 在 Screen 2 與 Screen 3 中，麵包屑第一層「行銷中心」連結至 `marketing-landing`（LP 儀表板）。使用者預期點擊「行銷中心」回到行銷中心的頂層導覽（例如優惠活動），而非 LP 儀表板。
- **根本原因：** Breadcrumb 的第一層 `{ label: '行銷中心', page: 'marketing-landing' }` 誤用 LP 儀表板 ID 作為連結目標。
- **建議修正：** 第一層麵包屑連結至 `coupon-list`（行銷中心入口）或設為純文字（不可點擊）：
  ```jsx
  breadcrumbs={['行銷中心', { label: '一頁式商店', page: 'marketing-landing' }, '建立']}
  ```

**問題 Mi-02：Slug 自動產生邏輯對中文產品名稱產生無效值**

- **影響任務：** T3
- **位置：** `html/行銷中心.html:1590–1596`
- **問題描述：** 選擇產品後，系統根據產品名稱自動帶入 Slug。但中文產品名如「水光精華 EX」處理後只剩 `-ex`（以連字號開頭，不合格）。使用者看到 Slug 欄位自動帶入 `-ex` 會困惑這是否正確。
- **根本原因：** `v.toLowerCase().replace(/\s+/g, '-').replace(/[^a-z0-9-]/g, '')` 會把所有中文字移除，若產品名稱幾乎全為中文，結果就是空字串或只剩連字號前綴。
- **建議修正：** 在自動帶入 Slug 前加一層清理：去除前後連字號、若結果少於 3 字元則清空（讓使用者自行填寫）：
  ```javascript
  const sug = v.toLowerCase().replace(/\s+/g, '-').replace(/[^a-z0-9-]/g, '').replace(/^-+|-+$/g, '');
  if (sug.length >= 3) { set('slug', sug); checkSlug(sug); }
  ```

**問題 Mi-03：點擊商店名稱回到同一頁（儀表板），未進入詳情**

- **影響任務：** T2
- **位置：** `html/行銷中心.html:1517`
- **問題描述：** 規格要求點擊商店名稱進入「數據統計頁」（Screen 9）。Prototype 中點擊名稱等於導航至 `marketing-landing`（即現在這個頁面），頁面刷新但位置不變。使用者多次點擊後會以為名稱根本不能點。
- **根本原因：** 數據統計頁（Screen 9/10）未納入此次 Prototype 範圍，以 `marketing-landing` 作為 fallback。
- **建議修正：** 若詳情頁 Stub 短期不做，至少移除 `cursor: pointer` 並去掉 `onClick`，讓名稱視覺上不可點擊，避免誤導。

**問題 Mi-04：Step 3「返回加購設定」按鈕導至 Step 1 基礎設定**

- **影響任務：** T5
- **位置：** `html/行銷中心.html:1954`
- **問題描述：** Step 4 左下角按鈕標示「返回加購設定」，但 `onClick` 導航至 `lp-create-step1`（基礎設定）。使用者點下後看到完全不相關的頁面，造成明確困惑。
- **根本原因：** `lp-create-step3`（加購設定）尚未實作，連結以 Step 1 代替，但按鈕文字未同步修改。
- **建議修正：** 改按鈕文字為「返回基礎設定」，與實際導向一致；或隱藏此按鈕直到 Step 3 完成。

**問題 Mi-05：篩選列缺少「建立時間」排序選項**

- **影響任務：** T2
- **位置：** `html/行銷中心.html:1476`
- **問題描述：** UX Spec 排序欄位指定 CVR / GMV / PV / **建立時間** 四個選項。Prototype 只有三個（CVR/PV/GMV），缺少「建立時間」。
- **根本原因：** mock 資料無 `createdAt` 欄位，未實作此排序維度。
- **建議修正：** LP_DATA 各筆加入 `createdAt` 欄位，並在工具列新增「建立時間」排序按鈕。

## 五、受測者語錄

以下為根據實際 UI 文字與操作流程推演，非專業後台使用者最可能說出的話：

1. 「左邊有個『進階』小標籤，這是要另外付費才能用的嗎？還是我現在就能點進去？」
   — 看到 Sidebar 的「一頁式商店」旁橘色 進階 badge 時

2. 「有一個獎杯，應該是它今天最好，但我不確定 CVR 是什麼意思，是在說轉換率嗎？」
   — 瀏覽儀表板，嘗試回答哪個商店表現最好

3. 「我選了產品『水光精華 EX』，但下面的網址自動帶入了 '-ex'，這是對的嗎？我不知道要不要改掉。」
   — 在 Step 1 選完主推產品後看到 Slug 欄位

4. 「我按了『下一步：設計頁面』，結果跑到了『預覽與發布』，我是不是漏掉什麼了？」
   — Step 1 填完點下一步後跳至 Step 4

5. 「這個刪除按鈕是灰色的，我點了好幾次都沒反應，是壞掉了嗎？」
   — 嘗試刪除啟用中商店時

6. 「商店已成功發布了，我想去『廣告小幫手』追蹤，但我點了沒有反應。」
   — 發布後嘗試點擊廣告小幫手連結

7. 「我點了『返回加購設定』，但怎麼跑回去填名稱和產品那個頁面了？」
   — Step 4 點擊返回按鈕後
