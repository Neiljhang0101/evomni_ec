## Part 1 — Design Brief

### Product Vision

Evomni 行銷中心的優惠活動模組讓商家能快速建立、管理各種促銷活動，包含優惠券和滿額/滿件優惠。本次補齊的 7 個欄位與表單，聚焦在讓商家能更精細控制優惠條件（最高折扣上限、排除特定產品、指定發放對象）、建立贈品活動（訂單滿額或滿件即送指定贈品至購物車），並提供進階電商包的限時折扣與組合優惠（紅配綠）建立流程，讓商家可以製造購買緊迫感並帶動交叉銷售。

### Target User

**Primary:** 電商後台管理員（商家運營人員），熟悉促銷活動設定，需要快速建立並上線優惠，避免設定錯誤導致超額折扣或濫用。

### Design Goals

| # | 目標 | 如何驗證 |
|---|---|---|
| 1 | 商家設定百分比折扣時，能在同一畫面找到並填寫「最高折扣上限」欄位 | 可用性測試：要求商家建立一張「9 折但最多折 NT$500」的優惠券，觀察是否不需提示即可完成 |
| 2 | 商家設定自動發放券時，能選擇「滿額贈券」並設定發放對象（全體/等級/標籤） | 可用性測試：要求商家建立一張「訂單滿 NT$3,000 後自動發送給所有等級二以上會員」的券 |
| 3 | 商家能建立限時折扣活動並開啟前台倒數計時顯示，且清楚理解該開關的效果 | 觀察商家是否主動開啟倒數計時開關並確認 Tooltip 說明 |
| 4 | 商家能建立紅配綠組合優惠，正確設定 A 組、B 組商品與折扣條件，不混淆兩組的角色 | 要求商家建立「上衣 + 褲子各一件折 NT$200」活動，觀察是否正確填寫 A/B 兩組 |
| 5 | 商家能建立贈品活動，正確選擇滿額或滿件條件，指定贈品產品，並理解庫存不足時兩種處理方式的差異 | 要求商家建立「訂單滿 NT$1,500 自動送贈品」並設定庫存不足時「自動暫停」，觀察是否不需提示即可完成且能說明兩個選項的區別 |

### Design Principles

1. **情境揭露** — 只在相關條件成立時顯示欄位（例：百分比折扣才顯示上限欄位；自動發放才顯示發放對象）
2. **防呆優先** — 數值欄位需提供預設值與合理範圍，避免商家輸入超出邊界的值
3. **說明就在旁邊** — 每個非直覺欄位都要有 Tooltip 或 helper text，不讓商家猜測

### Design System Reference

- **Figma Library:** Evomni Design System（Element Plus 為基底）
- **Component framework:** Element Plus（`<el-input>`, `<el-select>`, `<el-switch>`, `<el-radio-group>`, `<el-input-number>`）
- **Icon set:** 與現有後台一致
- **Font:** Noto Sans TC

### Accessibility Requirements

- **Target WCAG level:** AA
- **Known constraints:**
  - 所有表單欄位必須可透過鍵盤操作（Tab / Shift+Tab）
  - 錯誤提示需出現在欄位旁，不僅依靠顏色區分

### Hard Constraints

- 禁止使用 Emoji 字元
- UI 標籤不得使用英文縮寫（如 SKU、RMA），需以白話中文標示並在括號內附縮寫
- 進階電商包功能必須顯示「進階電商包專屬」鎖定提示，不得直接隱藏

### Out of Scope

- 產品加價購設定介面（PRD 尚無詳細 UI 規格）
- 產品組合價設定介面（PRD 尚無詳細 UI 規格）
- 多階梯滿額折扣 UI（PRD 尚無詳細 UI 規格）
- 前台購物車優惠進度提示（本次只補後台）

### Screen 1：優惠券表單 — 基本設定 Tab（補充最高折扣上限）

**Purpose:** 商家設定優惠券的基本折扣方式，選擇百分比折扣時可另設金額上限，避免高消費訂單折扣金額過大。
**Entry points:** 優惠券管理列表點擊「新增優惠券」或表格列「編輯」後，切換至「基本設定」Tab。
**Primary user goal:** 建立一張有折扣上限的百分比優惠券。

#### Information Hierarchy

```
H1: 新增優惠券（或「編輯優惠券」）
Tab Bar: 基本設定 / 使用條件 / 發放設定（目前在「基本設定」）
表單區塊標題: 基本設定
主要 CTA: 儲存（固定底部 Bar）
次要 CTA: 取消
關鍵新增欄位: 最高折扣上限（僅百分比折扣時顯示）
```

#### Actual Copy

**Page / Section Headings**
- 頁面標題：`新增優惠券`（編輯時：`編輯優惠券`）
- 分頁標題：`基本設定`

**Button Labels**
- 主要操作：`儲存`
- 次要操作：`取消`
- 優惠代碼旁：`隨機產生`

**Form Fields**

| 欄位標籤 | 預填 / 佔位文字 | 說明文字（欄位下方） | 驗證錯誤提示 |
|---|---|---|---|
| 優惠券名稱 | `例：夏日購物節折扣` | 此名稱會顯示於顧客的優惠券清單，最多 60 字。 | 此欄位為必填，最多 60 字 |
| 優惠券類型 | —（Radio 選項） | — | 請選擇一種優惠券類型 |
| 折扣方式 | —（Select 選項） | — | 請選擇折扣方式 |
| 折扣數值（百分比） | `例：20`（輸入 20 = 打八折） | 輸入 1–99 的整數；輸入 20 代表打八折（折扣 20%）。 | 請輸入 1 至 99 的整數 |
| 折扣數值（固定金額） | `例：200` | 最少 NT$1。 | 請輸入折扣金額，最少 NT$1 |
| **最高折扣上限（新增）** | `例：500` | 選填。設定後，折扣金額最高不超過此上限，即使訂單金額很高也適用。 | 請輸入大於 0 的整數 |
| 優惠代碼 | `例：SUMMER20` | 英文字母與數字，4–20 碼。留空將自動產生隨機代碼。 | 此代碼已被使用，請更換 |

**Tooltip 內容（最高折扣上限）**
> 範例：設定九折且上限 NT$500，即使消費者購買 NT$10,000 的商品，最多只折 NT$500，不會無限制放大折扣。

**欄位顯示/隱藏規則**
- 「折扣方式」選擇「百分比折扣」時，顯示「最高折扣上限」欄位
- 「折扣方式」選擇「固定金額折扣」或「免運費」時，隱藏「最高折扣上限」欄位
- 「折扣方式」選擇「免運費」時，隱藏「折扣數值」欄位

**Toast / Notification Messages**

| 觸發時機 | 訊息文字 |
|---|---|
| 儲存成功（新增） | `優惠券已建立` |
| 儲存成功（編輯） | `優惠券已更新` |
| 優惠代碼重複 | `此代碼已被使用，請更換或點擊「隨機產生」` |

#### Component List

| 元件名稱 | 規格 | 互動行為 |
|---|---|---|
| Tab Bar | 3 個 Tab：基本設定 / 使用條件 / 發放設定 | 點擊 Tab 切換內容區塊；未儲存的資料在切換 Tab 時保留 |
| Radio Group（優惠券類型） | 折扣代碼 / 自動發放券 | 選擇後決定「發放設定」Tab 的內容 |
| Select（折扣方式） | 百分比折扣 / 固定金額折扣 / 免運費 | 選擇後動態顯示/隱藏「最高折扣上限」和「折扣數值」 |
| NumberInput（折扣數值） | 百分比：1–99；固定金額：min 1 | 輸入後即時驗證範圍 |
| NumberInput（最高折扣上限）| min 1，選填，預設空白 | 僅百分比折扣時顯示；旁有 Tooltip |
| TextInput + Button（優惠代碼）| 最多 20 字元，英數字 | 點擊「隨機產生」填入 8 碼隨機英數；失焦後驗證唯一性 |
| Fixed Action Bar | 儲存（主要）/ 取消（次要） | 固定於頁面底部；儲存前驗證所有必填欄位 |

#### Screen States

| 狀態 | 觸發條件 | 呈現方式 |
|---|---|---|
| 新增模式 | 點擊「新增優惠券」 | 所有欄位空白，預設「折扣方式：百分比折扣」 |
| 編輯模式 | 點擊現有優惠券「編輯」 | 帶入既有資料；有效期已過的券顯示唯讀 Banner |
| 百分比折扣選中 | 折扣方式選「百分比折扣」 | 顯示「最高折扣上限」欄位 |
| 非百分比折扣 | 折扣方式選其他 | 隱藏「最高折扣上限」欄位 |
| 驗證錯誤 | 點擊「儲存」但有必填未填 | 欄位下方顯示紅色錯誤提示文字，欄位框線轉紅色 |
| 代碼重複 | 優惠代碼失焦且後端驗證不通過 | 欄位下方顯示「此代碼已被使用，請更換」 |

#### Interaction Annotations

| 使用者動作 | 結果 |
|---|---|
| 選擇「百分比折扣」 | 「最高折扣上限」欄位以動畫淡入顯示 |
| 選擇其他折扣方式 | 「最高折扣上限」欄位以動畫淡出隱藏，清空暫存值 |
| 點擊 Tooltip 問號 | 展開說明文字氣泡 |
| 點擊「隨機產生」 | 填入 8 碼隨機英數大寫代碼，並即時呼叫後端驗證唯一性 |
| 點擊「儲存」 | 表單驗證 → 成功則回到列表並顯示 Toast；失敗則定位至第一個錯誤欄位 |

### Screen 2：優惠券表單 — 使用條件 Tab（補充排除產品欄位）

**Purpose:** 商家設定此優惠券的使用限制，包含最低消費金額、適用產品範圍，以及明確排除不參與此優惠的特定產品。
**Entry points:** 優惠券表單 Tab Bar 切換至「使用條件」。
**Primary user goal:** 指定哪些產品不能使用此優惠券。

#### Information Hierarchy

```
H1: 新增優惠券（或「編輯優惠券」）
Tab Bar: 基本設定 / 使用條件（目前） / 發放設定
表單區塊標題: 使用條件
關鍵新增欄位: 排除產品（不適用此券的產品）
次要欄位: 最低訂單金額 / 適用產品範圍 / 是否可疊加
```

#### Actual Copy

**Form Fields**

| 欄位標籤 | 預填 / 佔位文字 | 說明文字 | 驗證錯誤提示 |
|---|---|---|---|
| 最低訂單金額 | `0` | 消費者訂單金額須達到此金額才可使用本券。填 0 或留空代表無最低消費限制。 | — |
| 適用產品範圍 | —（Radio：全部產品） | 選擇「指定分類」或「指定產品」後，下方出現搜尋選擇器。 | 請至少選擇一個分類或產品 |
| **排除產品（新增）** | `輸入產品名稱搜尋…` | 選填。此清單中的產品不適用本券，即使符合其他條件也無法使用。 | — |
| 可與其他優惠疊加 | —（Switch，預設關閉） | 開啟後，顧客結帳時可同時套用本券與其他促銷活動。 | — |

**Tooltip 內容（排除產品）**
> 常見用途：活動期間某些特定品項（例：限量新品、已有折扣商品）不希望再被優惠券折抵，可在此排除。

**Switch 標籤**
- 可與其他優惠疊加（開啟）：`已開啟 — 可與促銷活動同時使用`
- 可與其他優惠疊加（關閉）：`已關閉 — 不可與其他優惠同時使用`

#### Component List

| 元件名稱 | 規格 | 互動行為 |
|---|---|---|
| NumberInput（最低訂單金額） | min 0，前置 NT$ 標籤 | 數值型，不可輸入負數 |
| Radio Group（適用產品範圍） | 全部產品 / 指定分類 / 指定產品 | 選「指定」後顯示對應的搜尋多選器 |
| Select 多選（適用分類/產品） | 遠端搜尋，輸入關鍵字即時查詢 | 選擇後以 Tag 方式顯示，可逐一移除 |
| Select 多選（排除產品，新增） | 遠端搜尋，輸入產品名稱查詢 | 與適用產品選擇器相同樣式；選中的排除產品以不同顏色 Tag 顯示（灰色） |
| Switch（可疊加） | 預設 OFF | 切換後立即更新旁邊的說明文字 |

#### Screen States

| 狀態 | 觸發條件 | 呈現方式 |
|---|---|---|
| 全部產品（預設） | 適用範圍選「全部產品」 | 無搜尋選擇器；排除產品欄位顯示 |
| 指定分類/產品 | 適用範圍選「指定分類」或「指定產品」 | 顯示對應搜尋選擇器 |
| 已選排除產品 | 商家搜尋並選擇排除產品 | 以灰色 Tag 顯示選中產品，可點 X 移除 |
| 排除產品搜尋中 | 輸入關鍵字後等待結果 | 下拉選單顯示「搜尋中…」 |
| 排除產品無結果 | 搜尋關鍵字無符合產品 | 下拉選單顯示「找不到符合的產品」 |

#### Interaction Annotations

| 使用者動作 | 結果 |
|---|---|
| 在排除產品欄位輸入關鍵字 | 300ms 去抖後觸發 API 搜尋，結果顯示於下拉選單 |
| 點選搜尋結果中的產品 | 以 Tag 形式加入排除清單；同一產品不可重複選取 |
| 點擊 Tag 上的 X | 從排除清單移除該產品 |
| 適用範圍選「指定產品」且部分產品也在排除清單中 | 顯示警告提示：「有 N 件產品同時在適用範圍和排除清單中，排除設定優先」 |

### Screen 3：優惠券表單 — 發放設定 Tab（補充滿額贈券觸發 + 發放對象）

**Purpose:** 商家設定優惠券的發放方式（手動輸入代碼或自動發放），自動發放時設定觸發條件（含新增的「訂單滿額」選項）以及發放對象。
**Entry points:** 優惠券表單 Tab Bar 切換至「發放設定」。
**Primary user goal:** 設定「顧客訂單滿 NT$3,000 後自動發送下次使用優惠券」並指定發放給特定會員等級。

#### Information Hierarchy

```
H1: 新增優惠券（或「編輯優惠券」）
Tab Bar: 基本設定 / 使用條件 / 發放設定（目前）
主要決策點: 發放方式（手動代碼 vs. 自動發放）
自動發放子欄位（條件性顯示）:
  觸發發放時機（含新增的「訂單達指定金額」）
  發放對象（新增）
  優惠券有效天數
```

#### Actual Copy

**Section Heading**
- 區塊標題：`發放設定`

**Form Fields**

| 欄位標籤 | 預填 / 佔位文字 | 說明文字 | 驗證錯誤提示 |
|---|---|---|---|
| 發放方式 | —（Radio：手動輸入代碼） | — | 請選擇發放方式 |
| 優惠券代碼（手動時） | `例：SUMMER20` | 英文字母與數字，4–20 碼。留空將自動產生。 | 此代碼已被使用，請更換 |
| 全站使用次數上限（手動時）| `0` | 填 0 或留空表示無上限。達到上限後優惠券自動失效。 | 請輸入大於或等於 0 的整數 |

## Assumptions

> 假設：A 組、B 組各各買一件才享優惠，不支援「A 組買 2 件 + B 組買 1 件」等複雜組合。此假設依 PRD §6.2C 規格，如有更複雜需求請 PM 補充。

> 假設：滿額贈券的滿額條件，以 Layer 2 折後金額（非原價）為判斷基準，與 §8.5.4 的決策一致。

> 假設：限時折扣的倒數計時從 JavaScript 前端實作，後端只提供活動結束時間；前端計時精度為秒。

> 假設：排除產品搜尋的去抖時間（debounce）為 300ms，每次搜尋最多返回 20 筆結果。

> Copy 待確認：限時折扣活動的「活動說明」欄位說明文字需確認是否顯示於消費者前台，或僅商家看到。

> 假設：贈品活動的「滿額」條件數值，以 Layer 2 折後金額為判斷基準（與 §8.5.4 的決策一致），非訂單原價。

> 假設：贈品產品庫存的扣減時間點為訂單成立（付款成功）時，而非加入購物車時；若在加入購物車至付款期間庫存耗盡，訂單成立時依「庫存不足時」的設定處理。
