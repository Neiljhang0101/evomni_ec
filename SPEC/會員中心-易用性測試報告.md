## 三、問題清單（依嚴重程度排序）

---

### 🔴 Critical

#### C-01 個人資料頁（Screen 4）顯示 B2B 商業欄位，消費者個人資料功能完全缺失

- **影響任務：** T5（更新個人資料）
- **位置：** `html/會員中心.html`，`ProfilePage` 元件，第 582–650 行
- **問題描述：** 個人資料頁目前顯示「E-mail（作為登入帳號使用）」「密碼 / 變更密碼」「公司名稱」「聯絡人職稱」「聯絡人姓名」「聯絡人電話」，這是後台管理員的 B2B 欄位，不是消費者的 B2C 個人資料。消費者完全找不到可以更改姓名、生日、性別、頭像的入口；OTP 手機更改流程及 Email 更改流程均未實作。
- **根本原因：** ProfilePage 元件似乎是從後台管理員版本直接搬用，未依照 UX Spec Screen 4 重寫為消費者版本。
- **建議修正：** 依照 UX Spec Screen 4 完整重寫 ProfilePage，包含：圓形頭像上傳區（80×80px）、姓名輸入框（必填）、手機號碼唯讀欄位 + 更改手機 OTP 流程、Email 唯讀欄位 + 更改 Email 確認信流程、生日年月日三個下拉選單、性別 Radio（男/女/不透露）、預設收件地址快覽、儲存變更（Disabled 直到有修改）/ 取消按鈕。

### 🟠 Major

#### M-01 手機版底部 Tab 只有 4 項，「點數」主要入口遺失

- **影響任務：** T2（手機版使用者查看點數）
- **位置：** `html/會員中心.html`，CSS 第 200 行（`grid-template-columns: repeat(4, 1fr)`）、`BottomTabs` 元件第 426–430 行、`MORE_ITEMS` 陣列第 381–387 行
- **問題描述：** 手機版底部 Tab 目前只有 4 個（訂單、個人資料、優惠券、更多），CSS 也設定為 4 欄。「點數 / 購物金」被錯誤地放入「更多」底部彈窗，而非作為直接的第 3 個 Tab。UX Spec 明確要求 5 個 Tab：`訂單`、`個人資料`、`點數`、`優惠券`、`更多`；「更多」展開後應包含`心願清單`、`收件地址`、`推薦好友`、`帳號安全`、`登出`，不含點數。
- **根本原因：** BottomTabs 元件少了點數 Tab；CSS 欄數配置未更新；MORE_ITEMS 清單包含了不應在此的「點數 / 購物金」。
- **建議修正：** 將 CSS 改為 `grid-template-columns: repeat(5, 1fr)`；在 BottomTabs 中加入點數 Tab；從 MORE_ITEMS 移除 `points`。

#### M-02 新增地址表單為 inline 展開，非對話框；縣市/行政區未聯動

- **影響任務：** T3（新增地址）
- **位置：** `html/會員中心.html`，`AddressesPage` 元件第 683–713 行
- **問題描述：** 點擊「新增地址」後，表單以 inline 方式在頁面下方展開，而非 UX Spec 要求的半透明遮罩 Dialog。更嚴重的是，縣市選擇變更後，行政區選項**完全不更新**——無論選哪個縣市，行政區選項始終是「大安區」和「西屯區」。消費者選了台中市卻只能選台北的行政區，必然造成資料錯誤。
- **根本原因：** 縣市/行政區下拉未實作聯動邏輯，行政區選項為靜態硬編碼。
- **建議修正：** 實作城市 → 行政區聯動（可用簡易 Map 物件對應）；選縣市後行政區先 reset 為「請選擇行政區」再更新可選項；Dialog 樣式為加分項。

#### M-03 訂單列表缺少搜尋功能

- **影響任務：** T1（當訂單數量多時）
- **位置：** `html/會員中心.html`，`OrdersPage` 元件第 459–506 行
- **問題描述：** UX Spec Screen 2 要求訂單列表上方有搜尋輸入框，Placeholder 為「搜尋訂單編號或商品名稱」，輸入後即時篩選並可清除。目前 OrdersPage 完全沒有搜尋輸入框，消費者只能透過 Tab 篩選狀態，無法直接搜尋特定訂單編號或商品。
- **根本原因：** 搜尋輸入框未實作。
- **建議修正：** 在 Tab 列下方新增搜尋輸入框元件，支援即時篩選 ORDERS 陣列的 id 和 items 名稱；有輸入時顯示清除 icon。

#### M-04 刪除帳號流程僅一步確認，缺少第二步 Email 驗證

- **影響任務：** T4（安全操作驗證）
- **位置：** `html/會員中心.html`，`SecurityPage` 元件第 951–953 行
- **問題描述：** UX Spec 要求刪除帳號為兩步驟流程：步驟一彈出 EvomniAlert 確認列點（個人資料/點數/優惠券/心願清單將移除），步驟二需輸入帳號 Email 確認。目前僅實作一次 EvomniAlert 點擊即可完成申請，缺少 Email 輸入的二次驗證。對不可逆操作而言，此安全防線缺失風險高。
- **根本原因：** 僅實作步驟一，步驟二 Email 確認表單未開發。
- **建議修正：** 步驟一 EvomniAlert `onConfirm` 後，觸發步驟二：顯示 Email 輸入對話框（或第二個 EvomniAlert），輸入的 Email 須與帳號 Email 相符才可送出刪除申請。

### 🟡 Minor

#### m-01 取消訂單 EvomniAlert 說明文字不完整，缺少退款說明

- **影響任務：** T1
- **位置：** `html/會員中心.html` 第 497 行
- **問題描述：** 程式碼的取消訂單說明文字為「訂單取消後無法恢復。」，UX Spec 要求：「訂單取消後無法恢復，若已付款將依退款政策處理。」對於已付款訂單，消費者看不到退款相關說明會感到困惑。
- **根本原因：** 說明文字不完整。
- **建議修正：** 將 `desc` 改為 `'訂單取消後無法恢復，若已付款將依退款政策處理。'`

#### m-02 點數異動明細欄位標題「點數」應為「點數變化」，篩選選項不完整

- **影響任務：** T2
- **位置：** `html/會員中心.html` 第 746、742–743 行
- **問題描述：** 表格欄位標題寫作 `點數`，UX Spec 要求 `點數變化`；類型篩選缺少「點數到期」選項；時間篩選缺少「自訂區間」選項。
- **根本原因：** 欄位標題與篩選選項未完整對應 Spec。
- **建議修正：** 將表頭 `<th>點數</th>` 改為 `<th>點數變化</th>`；補充「點數到期」類型選項與「自訂區間」時間選項。

#### m-03 優惠券「已使用」卡片的訂單連結無法點擊

- **影響任務：** T6
- **位置：** `html/會員中心.html`，`CpnRow` 元件第 773 行
- **問題描述：** 已使用優惠券卡片顯示「已於 YYYY/MM/DD 使用　訂單 #EC-XXXX」，但文字為靜態顯示，無點擊行為。UX Spec 要求「查看訂單 #EC-XXXX」為可點擊連結，導向訂單詳情頁。
- **根本原因：** 訂單連結未實作為可點擊元件。
- **建議修正：** 改為 `<button>` 或 `<a>` 元件並呼叫 onNav('order_detail')。

#### m-04 密碼輸入框缺少顯示/隱藏切換圖示

- **影響任務：** T4（修改密碼）
- **位置：** `html/會員中心.html`，`SecurityPage` 第 900–905 行
- **問題描述：** 三個密碼欄位均為標準 `type="password"`，無眼睛圖示可切換顯示/隱藏。UX Spec 要求密碼輸入框需有顯示/隱藏切換功能。消費者輸入後無法確認密碼是否正確。
- **根本原因：** 密碼顯示切換功能未實作。
- **建議修正：** 為三個密碼欄位加入眼睛圖示按鈕，點擊可切換 `type="password"` 與 `type="text"`。

#### m-05 優惠券頁各 Tab 空狀態未實作

- **影響任務：** T6
- **位置：** `html/會員中心.html`，`CouponsPage` 第 785–787 行
- **問題描述：** 目前優惠券頁面直接 `.map()` 渲染，若某 Tab 的陣列為空，顯示區域為空白，沒有任何說明。UX Spec 要求各 Tab 空狀態有獨立訊息（可使用：「目前沒有可使用的優惠券」；已使用：「還沒有使用過的優惠券紀錄」；已過期：「沒有過期的優惠券紀錄」）。
- **根本原因：** 空陣列判斷與 Empty State 元件未套用。
- **建議修正：** 各 Tab 加上 `COUPONS[tab].length === 0 ? <EmptySt> : ...` 判斷。

#### m-06 帳號安全「第三方帳號綁定」缺少說明文字

- **根本原因：** 說明文字未加入。

#### m-07 帳號安全「刪除帳號」說明文字缺少「收件地址」

## 五、受測者語錄

以下語錄為根據 UI 實際內容模擬非專業消費者的反應：

1. 「等等，這個『個人資料』頁面為什麼要我填公司名稱和聯絡人職稱？我只是想改一下我的姓名和生日，這應該是我個人的資料管理，不是公司的吧？」

2. 「我在手機上，底部有訂單、個人資料、優惠券、更多，但是我要看點數的話要去哪裡找？我點了『更多』才發現點數在這邊，感覺繞了一圈。」

3. 「新增收件地址的時候，我選了台中市，但是行政區的下拉沒有變，還是顯示大安區和西屯區，這好像是台北的區，我怎麼選到台中的行政區？」

4. 「取消訂單的提示說『訂單取消後無法恢復』，但如果我已經付款了，錢要怎麼退？這邊沒說清楚，我不確定要不要按確認。」

5. 「刪除帳號這麼重要的事，只要按一下確認就完成了，感覺這樣不太安全。就算有 30 天冷靜期，還是感覺輕易了一點，沒有讓我再確認一次身份的步驟。」
