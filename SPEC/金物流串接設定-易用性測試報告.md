## 三、問題清單（依嚴重程度排序）

### 🔴 P0 — Critical

**Issue C-01：廠商類型 Tag 缺少類型識別符**

- **影響任務：** T1、T2（所有廠商設定任務）
- **位置：** `components/SharedUI.jsx:188` — `VendorTypeTag` 函數
- **問題描述：** UX Spec 明確要求廠商類型 Tag 需含 emoji 識別符（`🤖 全自動`、`🔧 設定即用`、`🤝 半人工`、`👤 人工處理`），以讓非技術用戶一眼分辨廠商操作模式。目前 Prototype Tag 文字為純文字，無視覺區隔符號。用戶看到「全自動」、「人工處理」等標籤時，若沒有對應圖示，需要逐字閱讀才能理解差異。
- **根本原因：** `VendorTypeTag` 實作時省略了 emoji 前綴。
- **建議修正：**
  ```jsx
  // SharedUI.jsx:188
  const emojiMap = { '全自動': '🤖', '設定即用': '🔧', '半人工': '🤝', '人工處理': '👤' };
  // 在 return 時：
  return <span ...>{emojiMap[type] || ''} {type}</span>;
  ```

### 🟠 P1 — Major

**Issue M-01：Hub 頁架構改為 Left Nav，Tab Count Badge 消失**

- **影響任務：** T1（Design Goal #2：管理員能一眼看出哪些廠商連線異常）
- **位置：** `html/金物流串接設定.html:131-138`（左側 nav），`components/HubScreen.jsx:74-75`（原 VendorCategoryPanel 計數邏輯）
- **問題描述：** UX Spec Screen 1 設計 Tab Bar（銀行金流/第三方金流/第三方物流）並在每個 Tab 顯示「已啟用 N 家」數量 Badge。Prototype 改為 160px 左側垂直 Nav，沒有任何數量指示。用戶切換分類時無法快速掌握各分類的串接健康度。
- **根本原因：** 設計決策——以 Left Nav 容納廠商類別與功能設定兩個群組，但未補足 Left Nav 項目的數量提示。
- **建議修正：** 在每個廠商分類 Nav Item 後加上已啟用數量 `(N)` 或用顏色區分（若有連線異常則 Nav Item 文字變紅）。

**Issue M-02：廠商資訊 Section 預設收起，申請連結隱藏**

- **影響任務：** T2（首次設定廠商時找不到申請帳號引導）
- **位置：** `components/VendorAutoScreen.jsx:118` — `<SectionCard title="廠商資訊" collapsible defaultOpen={false}>`
- **問題描述：** UX Spec 明確標注廠商資訊 Section 「Collapse，**預設展開**」。目前 `defaultOpen={false}` 導致「前往綠界科技申請商家帳號」和「查看 API 串接文件」兩個關鍵引導連結在進入頁面時完全隱藏。對於還沒有申請商家帳號的首次用戶，這是主要卡點。
- **根本原因：** `defaultOpen={false}` 設定錯誤。
- **建議修正：** 改為 `defaultOpen={true}`（或移除 `defaultOpen`，`SectionCard` 預設值為 `true`）。

**Issue M-03：重量費率空狀態未實作（已啟用無費率階梯時）**

- **影響任務：** T4 延伸（用戶啟用重量運費後立即想新增費率）
- **位置：** `components/FreightSettingsScreen.jsx:250-251`
- **問題描述：** UX Spec 要求「已啟用但 Table 無任何行」時顯示 Empty State（Headline: 「尚未設定任何費率階梯」、CTA: `+ 新增費率階梯`）。但 Prototype 初始化時預設有一行 `[{ id: 1, min: 0, max: '', ... }]`（`useState` 初值），不會觸發空狀態；且當唯一一行時刪除按鈕已 disabled，所以在實務上永遠不會出現空狀態。實際上空狀態規格被初始化邏輯繞過。
- **根本原因：** `tiers` 初始值預帶一行，遮蓋了空狀態。
- **建議修正：** 初始值改為 `[]`，啟用開關 ON 後若無行則顯示 Empty State + CTA；刪除最後一行時不 disable 而是改為顯示 Empty State。

**Issue M-04：Webhook URL 清單狀態語意混淆**

- **影響任務：** T6
- **位置：** `components/UtilScreens.jsx:200`
- **問題描述：** URL 清單的狀態欄位使用 `StatusBadge` 並傳入 `'已啟用'` 或 `'連線異常'`：
  ```jsx
  <StatusBadge status={w.status === '正常' ? '已啟用' : '連線異常'} />
  ```
  「已啟用」是廠商串接的狀態詞彙，用在 Webhook URL 健康狀態欄中，用戶會困惑「Webhook 已啟用是什麼意思？」UX Spec 此欄位應顯示「正常」/「異常」。
- **根本原因：** 直接複用了廠商狀態的 StatusBadge，未為 Webhook 狀態建立獨立的對照值。
- **建議修正：** 改為直接用 `w.status`（「正常」/「連線異常」），或在 `StatusBadge` 的 `map` 中補上「正常」→ 綠色對應。

### 🟡 P2 — Minor

**Issue N-01：取消按鈕文字不隨有無變更而切換**

- **位置：** `components/SharedUI.jsx:174` — `FixedBar`
- **問題描述：** UX Spec 說「有未儲存變更時取消顯示為『放棄變更』」。`FixedBar` 按鈕文字固定為「取消」，不管是否有 dirty state。用戶點「取消」後才彈 Dialog，會感到意外。
- **建議修正：** `FixedBar` 接收 `dirty` prop，按鈕文字依 dirty 狀態切換：`dirty ? '放棄變更' : '取消'`。

**Issue N-02：Hub 頁面標題與 Spec 不符**

- **位置：** `html/金物流串接設定.html:123`
- **問題描述：** UX Spec Screen 1 Page title: `串接設定`；Prototype PageHeader `title="金物流串接"`。對用戶影響輕微，但文件與 Spec 不一致。

**Issue N-03：搜尋空狀態缺少引導 Subtext**

- **位置：** `components/HubScreen.jsx:110`
- **問題描述：** UX Spec 空搜尋結果要求 Subtext「請確認廠商名稱後再搜尋」。Prototype 只有 Headline + 清除按鈕，無說明文字，用戶不確定是輸入錯誤還是系統問題。

**Issue N-04：環境切換 Radio 選項缺少 emoji**

- **位置：** `components/VendorAutoScreen.jsx:147-150`
- **問題描述：** UX Spec 選項文字為「🧪 測試模式（沙箱）」和「🚀 正式模式（Live）」。Prototype 選項無 emoji，測試/正式模式差異性在視覺上不夠顯著，Design Goal #5（測試與正式模式差異讓商家不會混淆）受影響。

**Issue N-05：電子發票/Webhook 管理頁缺少麵包屑**

- **位置：** `components/UtilScreens.jsx:26`、`176`
- **問題描述：** InvoiceScreen 和 WebhookScreen 使用 `<h2>` 作為標題，無麵包屑導覽。UX Spec Screen 6 Breadcrumb: `全域設定 / 金物流串接 / 電子發票設定`；Screen 7 Breadcrumb: `全域設定 / 金物流串接 / Webhook 管理`。用戶在頁面內無法確認自己在哪裡。

**Issue N-06：溫層/重量設定頁 H2 顏色與其他廠商設定頁 H1 風格不一致**

- **位置：** `components/FreightSettingsScreen.jsx:92`、`198`
- **問題描述：** TempRulesPanel 和 WeightRulesPanel 使用 `<h2 style={{ color: '#409EFF' }}>` 作為標題，其他廠商設定頁（VendorAutoScreen 等）使用 `<h1 style={{ color: '#303133' }}>` + PageHeader。視覺語言不一致，藍色 h2 像是 Section 標題而非頁面標題。建議改用 PageHeader 元件統一。

### 🔵 Enhancement

**Issue E-01：VendorSemiScreen 硬編碼 `isAdvanced = false`**

- **位置：** `components/VendorOtherScreens.jsx:280`
- **問題描述：** `const isAdvanced = false;` 使整個 BOXFUL 設定區域永遠 disabled + opacity 0.5。用戶看到 Banner 提示升級，但點 "了解升級方案" 只有 Toast，沒有跳轉。Prototype 演示效果受限，PM/設計師在 review 時可能誤以為功能未完成。

**Issue E-02：Hub 廠商 Card Grid 由 4 欄縮為 3 欄，資訊密度下降**

- **位置：** `components/HubScreen.jsx:114`
- **問題描述：** UX Spec Token Reference 明確標注「Card Grid columns（桌機）: 4 欄」。Prototype 因左側 Nav 佔用 160px 而改為 3 欄。在廠商較多的類別（第三方金流 7 張 Card），3 欄需多捲動一行。可考慮在 1280px+ 視窗下維持 4 欄（right panel 寬度仍足夠），或在 UX Spec 補充 2-column layout 決策。

## 五、受測者語錄（AI 模擬非專家用戶視角）

1. **進入 Hub 頁面**：「這裡看起來像是設定清單，但我不太懂這些廠商哪個是做什麼的——全自動、設定即用，有什麼差別？如果每個 Card 有個小圖示就好了。」

2. **看到連線異常 Card**：「哦，這個卡片頂部有條紅線，看起來不對勁。把滑鼠移上去，出現了一個訊息說『AUTH_FAILED 2026/05/10』——所以是金鑰錯誤？我先點進去看看。」

3. **廠商資訊 Section 預設收起**：「我點進來設定綠界，但我都還沒有帳號耶。要去哪裡申請？找了一下，發現有個叫『廠商資訊』的區塊，點開才有申請連結——這個不應該預設就讓我看到嗎？」

4. **環境切換至正式模式**：「『測試模式（沙箱）』和『正式模式（Live）』——我大概知道沙箱是測試用，但如果有個圖示告訴我哪個是正式就更放心了。點了正式模式，跳出一個確認視窗，還好有這個，不然我可能就直接切了。」

5. **Webhook URL 清單**：「這個表格說我的 URL 狀態是『已啟用』，但旁邊也有一個是『連線異常』……『已啟用』是正常嗎還是異常？有點分不清楚。」

6. **追蹤號上傳完成後**：「王小明那筆訂單有一個橘色的提示說已超過 48 小時，我知道要先處理它。輸入追蹤號之後按確認，又跳出一個『此操作不可逆』的警告——對，出貨通知一旦發出去就沒辦法收回了，這個提醒很重要。」

7. **重量運費空費率階梯**：「我開啟了重量運費，但下面的費率表裡已經有一行了……這行是空的，我要從這行開始填嗎？我以為會有一個新增按鈕讓我從頭開始，有點不確定。」
