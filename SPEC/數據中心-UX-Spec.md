## Part 1 — Design Brief

### Product Vision

數據中心是 Evomni 電商後台的統一分析入口，讓商家老闆與行銷人員不再靠直覺做決策。透過白話文指標名稱、自動生成的文字結論、以及每個數字附帶「這為什麼重要」的 Tooltip，即使沒有數據背景的商家，也能一眼看懂今日業績、哪些產品在賺錢、哪些顧客值得維繫、哪個促銷沒人用——然後馬上採取行動。

### Target User

**Primary:** 商家老闆 / 商家管理員 — 每日確認業績狀況，需要快速掌握 GMV、訂單數等核心指標，不具備深度數據分析背景。

**Secondary:** 行銷人員 — 負責促銷規劃與廣告投放，需要 RFM 分群、漏斗數據、廣告 ROAS 等進階洞察來決策行銷資源配置。

### Design Goals

| # | Goal | How to test |
|---|---|---|
| 1 | 商家進入數據中心後 30 秒內，不需要教學就能找到今日 GMV | 觀察用戶找到「總銷售金額」卡片所需時間 |
| 2 | 商家看完一個圖表後，能說出一個可採取的行動（不是只說「數字變了」）| 訪談用戶讀完時段熱力圖後，問：「你接下來會做什麼？」|
| 3 | 啟航方案用戶看到鎖定功能時，能理解「升級後能獲得什麼」（不是感到被拒絕）| 觀察是否主動詢問升級方式，而非直接離開頁面 |
| 4 | 行銷人員能在 2 分鐘內找到特定 RFM 分群並觸發行銷旅程 | 計時觀察從進入「會員分析」到點擊「發送行銷旅程」的時間 |
| 5 | 商家點擊「匯出報表」後，不需要等待，能繼續進行其他操作 | 觀察點擊後頁面是否立即回應，用戶是否知道「下一步要做什麼」|

### Design Principles

1. **白話優先** — 所有指標名稱、文字結論、Tooltip 都用商家老闆看得懂的語言，絕不出現技術術語（如「OLAP 快照」、「RFM Score」作為介面文字）
2. **圖表 + 文字結論缺一不可** — 每個圖表下方必須有自動生成的一句話白話建議，讓不懂看圖的用戶也能獲得洞察
3. **升級引導製造欲望，不製造挫折** — 進階功能用模糊預覽讓啟航用戶「看到有什麼但看不清」，而非空白或錯誤訊息
4. **行動導向** — 每個數據洞察必須有對應的「下一步操作」連結（如 RFM 分群→觸發旅程、滯銷產品→快速加入促銷）
5. **桌機優先** — 數據中心為桌機專用（最小 1440px），不要求手機最佳化

### Design System Reference

- **Figma Library:** Evomni Design System（Element Plus 為基礎）
- **Component framework:** Element Plus（`<el-card>`, `<el-tabs>`, `<el-date-picker>` 等）
- **Icon set:** Element Plus icons
- **Font:** Noto Sans TC, sans-serif

### Accessibility Requirements

- **Target WCAG level:** AA
- **Known constraints:**
  - 所有熱力圖顏色深淺不可只靠色彩區分，需搭配 Hover Tooltip 說明數字
  - 漲跌指標（紅綠色）必須同時有箭頭符號（↑↓）避免色盲用戶無法辨識
  - 最小可點擊區域 32px × 32px

### Hard Constraints

- ⚠️ 數據中心嚴禁直接查詢 OLTP 主庫，設計上**不應**有「即時刷新」按鈕，避免誤導商家期待即時數據
- ⚠️ 顧客 Email 在所有畫面一律部分遮蔽（`te***@gmail.com`）；設計中不可顯示完整 Email
- ⚠️ 毛利率 / 成本價欄位在 Figma 設計中需標注「僅擁有者角色可見」
- ⚠️ CMS 原生分頁（流量分析、SEO 報告、表單成效）不在本 PRD 範圍，設計時不需展開其內容
- ⚠️ 所有匯出功能一律非同步（背景產生→寄信），設計中不應出現「立即下載」的視覺暗示

### Out of Scope

- CMS 流量分析、SEO 報告、表單成效（CMS 原生功能，由 CMS 團隊維護）
- 手機版 / 響應式設計（桌機 1440px 以上專用）
- 數據中心的「設定」頁面（如時區設定，在全域設定模組）
- 顧客完整個資匯出（需擁有者角色，在匯出流程另行處理）

| # | Screen Name | Navigation path | Primary user goal |
|---|---|---|---|
| 1 | 數據中心 — 通用框架 | 後台左側導覽 → 數據中心 | 選擇時間範圍、切換電商四個分頁 |

### Screen 1: 數據中心 — 通用框架

**Purpose:** 提供所有電商分析分頁共用的頂部控制列，包含 Tab 切換、時間範圍選擇、資料說明。
**Entry points:** 後台左側導覽 → 數據中心
**Primary user goal:** 選擇查詢的時間範圍，進入想查看的分析分頁

#### Information Hierarchy

```
H1 (most prominent): 數據中心
H2 (secondary): [目前分頁名稱]
Primary CTA: 匯出報表
Secondary CTA: 顯示同期對照（Toggle）
Supporting info: 資料更新至 YYYY/MM/DD 00:30
```

#### Actual Copy

**Page / Section Headings**
- 頁面標題: `數據中心`
- Tab 標籤: `銷售概覽` / `產品分析` / `會員分析` / `行銷成效`
- 啟航方案進階 Tab 後綴: `行銷成效 🔒`

**Button Labels**
- Primary action: `匯出報表`
- Toggle 標籤: `顯示同期對照`

**時間範圍快選按鈕（由左至右）**
`今天` / `昨天` / `本週` / `本月` / `上月` / `近 7 天` / `近 30 天` / `近 90 天` / `自訂`

**Helper / Info Text**
- 資料更新說明: `資料更新至 YYYY/MM/DD 00:30`
- 今日資料附加說明: `（即時，5 分鐘快取）`

**Empty State**（所選時間範圍無任何訂單）
- Headline: `所選時間範圍內沒有資料`
- Subtext: `若剛開店，請等到有訂單後再查看。或嘗試選擇更長的時間範圍。`
- CTA label: `選擇其他時間範圍`

**Error States**
| Error condition | Message shown to user |
|---|---|
| 資料載入失敗 | `資料暫時無法載入，請稍後再試。若問題持續發生，請聯絡客服。` |
| 自訂時間範圍超過 365 天 | `最長可查詢 365 天的資料範圍，請縮短時間區間。` |
| 結束日期早於開始日期 | `結束日期必須晚於開始日期` |

**Loading State**
- 切換 Tab / 時間範圍時: 骨架屏（Skeleton 波紋效果），無文字

**Toast / Notification Messages**
| Trigger | Message |
|---|---|
| 點擊「匯出報表」 | `報表產生中，完成後將寄送至您的信箱 📧` |

#### Component List

| 元件名稱 | 規格 / 變體 | 互動行為 |
|---|---|---|
| Tab Bar | `<el-tabs>` 水平，4 個電商分頁；進階 Tab 顯示 🔒 icon | 點擊切換分頁；🔒 Tab 點擊彈出升級引導彈窗（Screen 6）|
| Date Range Picker | `<el-date-picker type="daterange">`；含快選按鈕列 | 選擇後頁面所有圖表同步更新（Skeleton 載入）|
| 快選按鈕組 | `<el-radio-group>` 按鈕式；9 個選項 | 點擊自動填入對應日期範圍 |
| 同期對照 Toggle | `<el-switch>`；標籤：「顯示同期對照」；預設關閉 | 開啟後圖表疊加上期虛線 |
| 匯出報表按鈕 | `<el-button class="!rounded-none">` Secondary 樣式 | 點擊顯示 Toast；背景非同步產生報表 |
| 資料更新時間 | 12px，`#909399`，頁面右上角 | 無互動 |

#### Screen States

| 狀態 | 觸發條件 | 呈現方式 |
|---|---|---|
| Default | 正常進入數據中心 | 銷售概覽 Tab 預設選中；顯示本月資料 |
| Loading | 切換 Tab 或時間範圍時 | 內容區 Skeleton 骨架屏 |

| User action | Result |
|---|---|

### Screen 2: 銷售概覽

**Purpose:** 讓商家一眼掌握業績現況，包含 GMV 等核心指標卡片、趨勢折線圖、付款方式分佈，以及進階的時段熱力圖與訂單來源分析。
**Entry points:** 數據中心 → 銷售概覽 Tab（預設分頁）
**Primary user goal:** 確認今日/本月業績是否達標，了解銷售趨勢高低峰

#### Information Hierarchy

```
H1 (most prominent): 四個核心指標卡片橫排（GMV 最左、字體最大）
H2 (secondary): 銷售趨勢折線圖（佔全寬，視覺重心）
H3 (tertiary): 付款方式圓餅圖（左）/ 時段熱力圖（右，進階）
Supporting info: 訂單來源分析表格（進階）
Primary CTA: 設定廣告花費（訂單來源區塊）
```

#### Actual Copy

**Page / Section Headings**
- 分頁標題: `銷售概覽`
- 區塊標題: `銷售趨勢` / `付款方式分佈` / `時段熱力圖` / `訂單來源分析`
