## 三、問題清單（依嚴重程度排序）

---

### 🔴 Critical 問題

**C-1：產品狀態缺少「已下架」選項**

- 影響：T-2（設定產品狀態）
- 位置：`AddEditProductPage.jsx` 第 195 行
- 描述：狀態 radio 只有三個選項 `公開 / 未公開 / 指定公開時間`，缺少 `已下架`。管理員無法在產品建立/編輯頁將產品設為下架狀態，只能依賴列表頁批次操作。
- 根本原因：實作時遺漏 `unpublished` 選項
- 建議修正：補齊第四個選項，並將所有 label 對齊 PRD：`已上架 / 草稿 / 已下架 / 已排程`

---

**C-2：狀態選項 Label 與 PRD 定義不一致**

- 影響：T-2
- 位置：`AddEditProductPage.jsx` 第 195 行
- 描述：選項文字為 `公開 / 未公開 / 指定公開時間`，PRD/UX Spec 定義應為 `已上架 / 草稿 / 已下架 / 已排程`。管理員看到「公開」不知道對應的是「已上架」還是「排程中」，造成認知混淆。
- 根本原因：前期開發時使用非規格術語，後續未更新
- 建議修正：Label 完全對齊 UX Spec 的狀態顯示文字

---

**C-3：「進階設定」Tab 不存在，溫層屬性無法被找到**

- 影響：T-4
- 位置：`AddEditProductPage.jsx` 第 92–96 行
- 描述：Tab 結構為 `基本設定 / 銷售資訊 / 相關產品 / 網頁SEO`。UX Spec 要求為 `基本設定 / 規格與定價 / 進階設定 / 網頁SEO`。「相關產品」Tab 的內容僅為 `相關產品設定（整合中）`，不包含任何進階設定欄位；溫層屬性（`tempType`）雖在 state 中宣告，但未在任何 Tab 中渲染輸出。
- 根本原因：Tab 名稱未更新、進階設定 Tab 內容尚未實作
- 建議修正：重命名 `銷售資訊` → `規格與定價`，`相關產品` → `進階設定`；並在進階設定 Tab 中實作溫層屬性 Radio Group 與 0元產品 Switch

---

### 🟠 Major 問題

**M-1：全站術語「產品」未統一為「產品」**

- 影響：所有 Screen
- 位置：`Sidebar.jsx`、`ProductListPage.jsx`、`AddEditProductPage.jsx`、`CategoryManagementPage.jsx`
- 描述：Sidebar 群組標籤（`產品中心 / 產品管理 / 產品分類`）、頁面 H1 標題（`產品管理`、`新增產品`）、麵包屑（`產品中心 > 產品管理`）、按鈕文字（`新增產品`、`刪除產品`）均使用「產品」，與 UX Spec 全面使用「產品」的規格不符，造成前後台用詞不一致。
- 根本原因：早期版本使用「產品」，UX Spec 更新後未同步套用至原始檔案
- 建議修正：全文替換（見下方 PRD 符合性檢查表）

---

**M-2：批次操作缺少「移動分類」選項**

- 影響：T-1（產品列表批次操作）
- 位置：`ProductListPage.jsx` 第 207–211 行
- 描述：批次操作 Dropdown 只有 `批次上架 / 批次下架 / 批次刪除`，缺少 PRD 明確要求的「移動分類」功能，管理員無法批次調整產品分類。
- 建議修正：新增「移動分類」Dropdown 選項，點擊後開啟分類選擇 Drawer

---

### 🟡 Minor 問題

**m-1：搜尋框 Placeholder 文字不符規格**

- 位置：`ProductListPage.jsx` 第 142 行
- 描述：目前為 `產品名稱、介紹、SKU`，UX Spec 規定為 `搜尋產品名稱或 SKU 編號`
- 建議修正：更新 placeholder 文字

---

**m-2：產品建立/編輯頁麵包屑結構錯誤**

- 位置：`AddEditProductPage.jsx` 第 134–138 行
- 描述：目前顯示 `產品 / 內容管理 / 產品內容`，UX Spec 規定為 `產品中心 > 產品管理 > 新增產品`（編輯模式為 `產品中心 > 產品管理 > 編輯：{產品名稱}`）
- 建議修正：更新麵包屑文字與結構

---

**m-3：刪除確認 Modal 按鈕文字不符**

- 位置：`ProductListPage.jsx` 第 125 行
- 描述：按鈕文字為 `刪除產品`，UX Spec 規定為 `刪除產品`
- 建議修正：更新按鈕文字

---

**m-4：CategoryManagementPage 使用「產品分類」術語**

- 位置：`CategoryManagementPage.jsx` 第 103 行
- 描述：Drawer 標題為 `新增產品分類 / 編輯產品分類`，應改為 `新增產品分類 / 編輯產品分類`
- 建議修正：更新 Drawer 標題文字

---

### 🔵 Enhancement 建議

**e-1：複製產品功能已實作，但視覺規格不符**

- 位置：`ProductListPage.jsx` 操作欄（第 284 行）
- 描述：操作欄已有文字按鈕「複製」，點擊後顯示 Toast `已複製「{產品名稱}」為草稿`，功能邏輯正確。但 PRD §3.1 規格要求為 icon 按鈕並附帶 Tooltip（`複製此產品`），目前為純文字按鈕，無 icon 也無 Tooltip。
- 建議修正：將文字按鈕改為 icon 按鈕（複製圖示），並加上 Tooltip：`複製此產品`

---

**e-2：載入中缺少 Skeleton Loader**

- 位置：所有 Screen
- 描述：目前所有頁面在模擬 API 等待時直接空白，PRD/UX Spec 要求顯示 Skeleton Loader（產品列表 5 列）
- 建議修正：加入 loading state 與對應的 Skeleton 元件

---

**e-3：缺少拖曳排序功能**

- 位置：`ProductListPage.jsx`
- 描述：PRD §3.1 Interaction Annotations 要求支援拖曳列調整 sort_order，目前不支援
- 建議修正：整合 drag-and-drop 排序（可用 @dnd-kit 或 SortableJS）
