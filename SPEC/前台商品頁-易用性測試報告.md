## 三、問題清單（依嚴重程度排序）

---

### 🔴 Critical

#### C-01 商品詳情頁：`descTab` state 宣告但整個 Tab 系統未實作

- **影響任務：** T4
- **位置：** `html/商品詳情.html` 第 310 行（state 宣告），整個 App JSX 無 Tab UI 渲染
- **問題描述：** UX Spec Screen 2 明確要求一組 Tab 系統（商品說明 / 成分表）讓消費者切換查看詳細說明與全成分列表。`const [descTab, setDescTab] = useState('desc')` 已宣告，但整個 App 的 JSX 中完全沒有使用此 state、沒有渲染 Tab 按鈕、沒有條件渲染 Tab 內容。消費者唯一能看到成分資訊的方式是捲動到頁面後段的 editorial 區塊，點開 `<details>` accordion，路徑冗長且不直覺。
- **根本原因：** Tab UI 規劃了 state 但未開發，可能是功能遺漏。
- **建議修正：** 在加購推薦區塊後、editorial 前插入 Tab 系統，渲染「商品說明」與「成分表」兩個 Tab，並使用 `descTab` state 切換內容。

---

### 🟠 Major

#### M-01 商品列表頁：grid inline style 阻擋手機 2 欄響應式

- **影響任務：** T6
- **位置：** `html/商品列表.html` 第 420 行
- **問題描述：** `.listing-grid` 的 inline style 包含 `gridTemplateColumns: 'repeat(3, 1fr)'`，CSS inline style 優先級高於任何 class 規則，導致 `rwd.css` 無法在手機（≤900px）覆蓋為 2 欄格。消費者在手機上看到 3 欄卡片，每張卡片寬度約 90px，商品名稱和圖片均難以辨讀。同樣問題：`mobile-cat-chips` 與 `listing-filter-bar` 有 `display: 'none'` inline style，防止 rwd.css 在手機顯示這些元件。
- **根本原因：** 開發時直接將響應式樣式寫入 inline style，阻擋了 CSS 媒體查詢覆蓋。
- **建議修正：** 移除 `.listing-grid` 的 inline `gridTemplateColumns`；在 `<style>` 中加入 `.listing-grid` 及媒體查詢；移除 mobile 元件的 `display: 'none'` inline，改用 CSS class 控制顯示/隱藏。

---

### 🟡 Minor

#### m-01 商品詳情頁：立即購買 EvomniAlert 缺少確認按鈕文字

- **位置：** `html/商品詳情.html` 第 356 行
- **問題描述：** `EvomniAlert.show(...)` 未設定 `confirmText`，確認按鈕使用系統預設文字。Spec 要求此處顯示「確認購買」。
- **建議修正：** 加入 `confirmText: '確認購買'`

#### m-02 商品詳情頁：main padding-bottom 80px，Spec 要求 100px

- **位置：** `html/商品詳情.html` 第 382 行
- **問題描述：** `padding: '40px var(--gutter) 80px'`。Spec 明確要求 100px 以防止手機底部 sticky CTA bar 遮蓋評論區最後一則。
- **建議修正：** 改為 100px

#### m-03 商品列表頁：收藏 icon 32×32px，Spec 要求 28×28px

- **位置：** `html/商品列表.html` 第 133 行
- **問題描述：** `width: 32, height: 32`。Spec 的元件清單指定 28×28px。
- **建議修正：** 改為 28×28

## 五、受測者語錄

以下語錄為根據 UI 實際狀態模擬非專業消費者的反應：

1. 「我想看這個商品用了哪些成分，點了加入購物車下面的加購區，再往下捲好長，才在一個小小的標題後面找到一個可以展開的選項，成分表怎麼這麼難找？」

2. 「我在手機上看，商品列表三欄好小，名字都看不清楚。其他購物網站手機版不都是兩欄嗎？」

3. 「到貨通知按下去有跳出提示說補貨會通知我，這個還蠻清楚的。不過我不確定會通知到哪裡，是 Email 還是手機？提示裡應該說清楚。」

4. 「我搜尋了『精華』，馬上就顯示出來幾個結果，這樣很方便，不用一個一個找。」

5. 「我要買，選了規格，然後按了立即購買，跳出一個確認視窗，但確認按鈕上面寫的不是很確定的文字，我稍微猶豫了一下才按。」
