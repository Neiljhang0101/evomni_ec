## Part 1 — Design Brief

### Product Vision

系統分頁管理讓後台管理者能在統一介面中查看、設定與維護所有電商網站必要的系統頁面（如隱私權政策、退款政策、404 錯誤頁等）。這些頁面過去散落在各功能模組中，缺乏統一管理入口。透過本模組，管理者在網站上線前能快速確認每個必填頁面的完成狀態，並一站式完成所有頁面的內容設定與發布。

### Target User

**Primary：** 電商後台管理員（一般管理員 / 超級管理員），負責在網站上線前完成法律合規頁面與系統功能頁面的設定，需快速掌握哪些頁面尚未設定

**Secondary：** 超級管理員，需要修改頁面路徑（網址）或管理維護模式白名單

### Design Goals

| # | Goal | How to test |
|---|---|---|
| 1 | 管理者能在 30 秒內找到所有未設定的必填頁面 | 易用性測試：請用戶找出哪些必填頁面尚未設定，計時並觀察操作路徑 |
| 2 | 管理者能完整設定一個法律合規頁面並發布，全程不需離開系統分頁模組 | 觀察任務完成率與操作路徑，確認無跳轉其他模組 |
| 3 | 管理者在儲存後能立刻確認頁面狀態已更新為「已設定」 | 觀察用戶是否需要手動重新整理頁面才能確認結果 |

### Design Principles

1. **完成度優先** — 核心任務是確保所有必填頁面設定完成，介面應讓未完成項目視覺上突出，不隱沒於清單中
2. **編輯不干擾清單** — 點擊編輯進入獨立編輯頁，避免行內展開導致清單可見性下降
3. **自動儲存降低焦慮** — 編輯過程持續顯示自動儲存狀態，讓使用者不擔心草稿遺失

### Design System Reference

- **Figma Library:** Evomni Design System
- **Component framework:** 依 Evomni 設計系統規範
- **Icon set:** Evomni Design System 內建圖示集
- **Font:** 依 Evomni Design System 規範

### Accessibility Requirements

- **Target WCAG level:** AA
- **Known constraints:**
  - 狀態標籤不可單純以顏色區分，須搭配圖示或文字
  - 鍵盤可完整操作（Tab 導覽、Enter 確認、Esc 取消對話框）
  - 富文字編輯器需支援鍵盤操作與所見即所得格式套用

### Hard Constraints

- 系統分頁不可刪除（僅能停用），因為這些頁面由系統預定義
- 一般管理員不可修改頁面路徑（網址），路徑欄位對其顯示為唯讀

### Out of Scope

- 主選單頁面管理（由選單管理模組處理）
- 商品詳情頁、部落格文章頁
- 使用者自行建立的自訂頁面

## Part 3 — Per-Screen UX Spec

### Screen 1: 系統分頁列表頁

**Purpose:** 提供所有系統分頁的一覽表，讓管理者快速掌握設定完成率並導覽至需要編輯的頁面。
**Entry points:** 後台側邊選單 > 網站設計 > 系統分頁
**Primary user goal:** 找出尚未設定的必填頁面，點擊進入編輯

#### Information Hierarchy

```
H1 (most prominent): 系統分頁
H2 (secondary): 設定完成度儀表板（X / Y 頁面已設定）
Primary CTA: 無（此為管理列表，主要動作為各列的「編輯」）
Supporting info: 篩選列、頁面清單表格、各列操作連結
```

#### Actual Copy

**Page / Section Headings**
- Page title: `系統分頁`
- Breadcrumb: `網站設計 / 系統分頁`
- Dashboard section heading: `設定完成度`

**Completion Dashboard**
- Progress label: `N / Y 頁面已設定`
- Warning text（有必填頁面未設定時）: `尚有 N 個必填頁面未設定，請在上線前完成`
- All complete text: `所有必填頁面已設定完成`

**Filter / Search Bar**
- Category filter label: `類型`
- Category options: `全部類型` / `法律合規` / `功能性頁面`
- Status filter label: `狀態`
- Status options: `全部狀態` / `已設定` / `未設定` / `停用`
- Search placeholder: `搜尋頁面名稱...`

**Table Column Headers**
- `頁面名稱` / `類型` / `狀態` / `路徑（網址）` / `最後更新` / `操作`

**Status Badge Labels**
| 狀態鍵值 | 顯示文字 | 色彩語意 |
|---|---|---|
| set | 已設定 | 成功（綠色） |
| unset | 未設定 | 警告（橙色，帶警告圖示） |
| disabled | 停用 | 中性（灰色） |

**Action Buttons（每列）**
- Primary: `編輯`
- Secondary: `預覽`（狀態為「未設定」時灰色不可點擊）

**Empty State（搜尋 / 篩選無結果）**
- Headline: `找不到符合條件的頁面`
- Subtext: `請調整篩選條件或清除搜尋關鍵字`
- CTA label: `清除篩選`

**Error States**
| 錯誤條件 | 顯示訊息 |
|---|---|
| 網路錯誤，資料載入失敗 | `頁面清單載入失敗，請重新整理後再試` 附「重新整理」按鈕 |

**Loading State**
- Loading text: skeleton only（無文字，模擬表格列排版寬度）

**Toast / Notification Messages**
| 觸發 | 訊息 |
|---|---|
| 從編輯頁儲存成功後返回 | `頁面已儲存並發布` |

#### Component List

| 元件名稱 | 規格 / 變體 | 互動行為 |
|---|---|---|
| 完成度儀表板 | 橫向進度條 + 文字摘要；警告樣式（有未設定必填頁）/ 成功樣式（全部完成） | 靜態顯示，無點擊行為 |
| 篩選下拉選單 | 類型篩選（3 選項）/ 狀態篩選（4 選項） | 選擇後即時過濾列表，不需另外點擊確認 |
| 搜尋輸入框 | 帶清除（X）按鈕 | 輸入後即時過濾，200ms debounce |
| 資料表格 | 固定表頭，N 列動態資料 | 點擊「編輯」導覽至 Screen 2 或 Screen 3 |
| 狀態標籤（已設定） | 綠色圓點 + 文字 | 靜態顯示 |
| 狀態標籤（未設定） | 橙色警告圖示 + 文字 | 靜態顯示 |
| 狀態標籤（停用） | 灰色圓點 + 文字 | 靜態顯示 |
| 「編輯」連結 | 文字連結或次要按鈕 | 點擊導覽至對應編輯頁 |
| 「預覽」連結 | 圖示按鈕；未設定時灰色不可點擊 | 點擊在新分頁開啟前台預覽 |
| 載入骨架 | 模擬表格列排版 | 資料載入中時顯示 |
| Toast | 成功樣式，3 秒後自動消失 | 從編輯頁儲存成功返回後顯示 |

#### Screen States

| 狀態 | 觸發條件 | 呈現方式 |
|---|---|---|
| 預設（有資料） | 正常載入 | 顯示所有頁面列表，儀表板正確計算完成率 |
| 有未設定必填頁面 | 部分必填頁面尚未設定 | 儀表板橙色警告文字 |
| 全部設定完成 | 所有頁面均已設定 | 儀表板綠色完成訊息 |
| 篩選後無結果 | 搜尋或篩選條件無符合項目 | 顯示空狀態（見上方 Empty State） |
| 載入中 | 首次進入頁面 | Skeleton 骨架佔位 |
| 網路錯誤 | API 請求失敗 | 錯誤訊息 + 重新整理按鈕 |

#### Interaction Annotations

| 使用者動作 | 結果 |
|---|---|
| 點擊「編輯」（法律合規類） | 導覽至 Screen 2：一般頁面編輯頁 |
| 點擊「編輯」（功能性頁面） | 導覽至 Screen 3：功能性頁面編輯頁 |
| 點擊「預覽」（已設定頁面） | 新分頁開啟前台頁面，頁面頂部顯示預覽提示條 |
| 切換類型或狀態篩選 | 即時過濾表格，不重新整理頁面 |
| 在搜尋框輸入文字 | 即時過濾頁面名稱，200ms debounce |
| 點擊「清除篩選」（空狀態） | 清空所有篩選條件，回到完整列表 |

### Screen 2: 頁面編輯頁（法律合規 / 一般內容）

**Purpose:** 讓管理者填寫與發布法律合規類系統頁面的完整內容，包含 SEO 設定。
**Entry points:** Screen 1 列表頁 > 點擊「編輯」（法律合規類頁面：隱私權政策、服務條款、退款政策等）
**Primary user goal:** 填寫頁面完整內容並點擊「儲存並發布」，使頁面在前台上線

#### Information Hierarchy

```
H1 (most prominent): 頁面名稱（如「隱私權政策」）
H2 (secondary): 富文字編輯器（主要操作區域）
Primary CTA: 儲存並發布
Secondary CTA: 預覽
Supporting info: 右側設定面板（頁面標題、路徑、狀態、SEO 描述）、自動儲存狀態、麵包屑
```

#### Actual Copy

**Page / Section Headings**
- Breadcrumb: `網站設計 / 系統分頁 / 隱私權政策`（以實際頁面名稱替換）
- Page title: `隱私權政策`（以實際頁面名稱替換）
- Main content section heading: `頁面內容`
- Settings sidebar heading: `頁面設定`
- SEO section heading: `搜尋引擎最佳化（搜尋優化）`

**Button Labels**
- Primary action: `儲存並發布`
- Secondary action: `預覽`
- Back navigation: `返回系統分頁`

**Form Fields**
| 欄位標籤 | 提示文字 | 說明文字 | 錯誤文字 |
|---|---|---|---|
| 頁面標題 | `輸入頁面標題` | `顯示於瀏覽器標籤與搜尋結果，建議 50–60 字元` | `頁面標題為必填欄位` |
| 路徑（網址） | `/privacy-policy` | `URL 路徑，部分頁面固定不可修改` | `路徑格式不正確，僅限英文、數字與連字號` |
| 狀態 | — | `關閉後訪客將無法看到此頁面` | — |
| 搜尋優化描述 | `輸入搜尋引擎顯示的頁面說明文字` | `建議 120–160 字元，目前已輸入 N 字元` | `搜尋優化描述不可超過 160 字元` |

**Multi-language Tabs（啟用多語系時顯示）**
- 繁體中文（預設，帶底線強調）
- English
- （依網站語系設定動態顯示）

**Auto-save Status（右側面板頂部）**
- Saving: `儲存中...`
- Saved: `草稿已自動儲存（HH:MM）`
- Unsaved changes: `有未儲存的變更`

**Empty State（編輯器空白時）**
- Headline: — （此畫面無頁面層級的空狀態，編輯器顯示 placeholder）
- Editor placeholder: `在此輸入頁面內容...`

**Error States**
| 錯誤條件 | 顯示訊息 |
|---|---|
| 儲存失敗（網路錯誤） | `儲存失敗，請確認網路連線後再試` |
| 路徑格式錯誤 | `路徑格式不正確，僅限英文、數字與連字號（-）` |
| 頁面標題未填 | `頁面標題為必填欄位` |
| 搜尋優化描述超過 160 字元 | `搜尋優化描述已超過上限（目前 N 字元，上限 160 字元）` |

**Loading State**
- 編輯器載入中: skeleton only（無文字，模擬編輯器高度佔位）

**Toast / Notification Messages**
| 觸發 | 訊息 |
|---|---|
| 手動儲存成功 | `頁面已儲存並發布` |
| 自動儲存成功 | 僅更新面板頂部的自動儲存狀態文字，不顯示 Toast |
| 儲存失敗 | `儲存失敗，請稍後再試` |

**Preview Banner（前台預覽分頁頂部）**
- `目前為預覽模式，此頁面對訪客不可見`

#### Component List

| 元件名稱 | 規格 / 變體 | 互動行為 |
|---|---|---|
| 麵包屑導覽 | 文字連結，最後一層不可點擊 | 點擊上層連結返回列表頁 |
| 多語系 Tab | 依啟用語系數量動態顯示，預設選中主要語系 | 點擊切換語系，編輯器與 SEO 欄位同步切換，各語系內容獨立 |
| 富文字編輯器 | 標準工具列（粗體、斜體、標題、清單、連結、圖片、表格） | 點擊工具列套用格式，支援鍵盤快捷鍵 |
| 頁面設定面板 | 固定右側欄（sticky sidebar） | 欄位輸入即時反映，不需點擊確認 |
| 文字輸入框（頁面標題） | 單行文字 | 失去焦點時觸發驗證 |
| 文字輸入框（路徑） | 單行文字；一般管理員為唯讀 | 超級管理員可編輯；一般管理員顯示唯讀樣式，游標為 not-allowed |
| 開關（狀態） | 啟用 / 停用，帶標籤文字 | 切換後本地即時反映，儲存後生效 |
| 文字區域（搜尋優化描述） | 多行文字，右下角顯示字元計數 N / 160 | 輸入即時更新字元計數，超過 160 字元顯示紅色計數 |
| 自動儲存狀態指示 | 文字標籤，3 種狀態：儲存中 / 已儲存（含時間）/ 有未儲存變更 | 靜態顯示 |
| 「儲存並發布」按鈕 | Primary 樣式，有 Loading 狀態 | 點擊觸發儲存，儲存中顯示 Loading 動畫 |
| 「預覽」按鈕 | Secondary 樣式 | 點擊在新分頁開啟前台預覽 |
| Toast 通知 | 成功 / 錯誤樣式，3 秒自動消失 | 手動儲存完成後顯示 |

#### Screen States

| 狀態 | 觸發條件 | 呈現方式 |
|---|---|---|
| 新頁面（無內容） | 點擊尚未設定頁面的「編輯」 | 編輯器空白顯示 placeholder，路徑欄位預填預設值 |
| 已有內容 | 點擊已設定頁面的「編輯」 | 編輯器載入現有內容，所有欄位填入既有資料 |
| 多語系（啟用多語時） | 網站設定啟用多語系 | 頁面頂部顯示語系 Tab |
| 自動儲存中 | 編輯後每 30 秒觸發 | 面板頂部顯示「儲存中...」 |
| 自動儲存完成 | 自動儲存 API 回應成功 | 顯示「草稿已自動儲存（HH:MM）」 |
| 手動儲存中 | 點擊「儲存並發布」 | 按鈕顯示 Loading，其他欄位鎖定 |
| 儲存成功 | 儲存 API 回應成功 | Toast 顯示成功訊息，返回列表頁 |
| 儲存失敗 | 儲存 API 回應失敗 | Toast 顯示錯誤訊息，停留在編輯頁 |
| 路徑唯讀（一般管理員） | 登入身份為一般管理員 | 路徑欄位唯讀樣式，帶提示文字「此欄位不可修改」 |
| 編輯器載入中 | 進入編輯頁，資料抓取中 | Skeleton 佔位 |

#### Interaction Annotations

| 使用者動作 | 結果 |
|---|---|
| 點擊「返回系統分頁」 | 若有未儲存變更，顯示確認對話框（確定離開 / 取消）；無變更則直接返回 |
| 點擊多語系 Tab | 切換語系，編輯器與搜尋優化描述欄位同步顯示對應語系內容 |
| 點擊「儲存並發布」 | 驗證表單；通過後呼叫儲存 API；成功後 Toast 通知並返回列表頁 |
| 點擊「預覽」 | 新分頁開啟前台頁面，頁面頂部顯示預覽提示條 |
| 30 秒後自動觸發 | 靜默儲存草稿，僅更新面板頂部狀態文字 |
| 一般管理員點擊路徑欄位 | 無反應（唯讀），游標顯示 not-allowed |
| 離開頁面時有未儲存變更 | 顯示確認對話框防止意外離頁 |

### Screen 3: 頁面編輯頁（功能性錯誤頁：404 / 500 / 維護中）

**Purpose:** 讓管理者自訂錯誤頁或維護頁的顯示文案、按鈕與視覺元素。
**Entry points:** Screen 1 列表頁 > 點擊「編輯」（404、500、維護中）
**Primary user goal:** 自訂錯誤 / 維護頁的標題、說明文字、按鈕與插圖，儲存並發布

> 本畫面繼承 Screen 2 的所有基本設定（右側設定面板：頁面標題、路徑、狀態開關、搜尋優化描述），並以「錯誤頁自訂設定」結構化表單取代富文字編輯器。以下僅記錄與 Screen 2 不同的部分。

> PRD 待補：PRD §4.3 缺失（文件從 §4.2 跳至 §4.4）。若有遺漏的功能規格，請補充至 PRD，本 UX Spec 以現有內容為準。

#### Information Hierarchy

```
H1 (most prominent): 頁面名稱（如「404 頁面」）
H2 (secondary): 錯誤頁自訂設定（結構化欄位區塊，取代富文字編輯器）
Primary CTA: 儲存並發布
Secondary CTA: 預覽
Supporting info: 右側設定面板（與 Screen 2 相同）
```

#### Actual Copy — 差異部分

**Additional Section Heading**
- `錯誤頁自訂設定`

| 欄位標籤 | 404 提示文字 | 500 提示文字 | 維護中提示文字 | 說明文字 |
