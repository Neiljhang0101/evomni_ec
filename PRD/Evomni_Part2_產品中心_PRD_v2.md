## 1. 產品概述

### 1.1 產品定位

**Evomni 產品中心**幫助電商店家管理員解決「產品資料維護耗時、多規格庫存難追蹤」的問題，讓他們能夠不需工程師介入，獨立完成產品上架、規格設定、庫存監控與批次作業。

---

### 1.2 目標用戶（Personas）

#### Persona A — 商家管理員（後台操作者）

| 項目 | 內容 |
|------|------|
| 角色 | 電商店家老闆或員工，負責日常產品維護 |
| 使用情境 | 每天早上進後台確認庫存、新增今日上架產品、處理缺貨通知 |
| 主要目標 | 1. 快速建立有多種顏色/尺寸的產品 SKU<br>2. 在活動前預排產品上下架時間<br>3. 即時掌握低庫存產品，及時補貨 |
| 最怕的事 | 建立 100 個產品卻要一個一個手動輸入；庫存扣成負數；誤刪有訂單的產品 |
| 技術熟悉度 | 一般用戶（會用 Excel，不懂程式） |

#### Persona B — 消費者（前台瀏覽者）

| 項目 | 內容 |
|------|------|
| 角色 | 一般網購消費者 |
| 使用情境 | 手機瀏覽產品頁，想確認尺寸是否有貨、想看清楚產品細節 |
| 主要目標 | 1. 快速找到想要的規格（顏色/尺寸）<br>2. 確認產品圖片細節再下單<br>3. 缺貨時訂閱貨到通知 |
| 最怕的事 | 選好規格後才發現無庫存；圖片太小看不清材質 |
| 技術熟悉度 | 一般用戶 |

---

### 1.3 核心使用流程（Top User Flows）

**Flow 1：建立多規格產品**
產品列表頁 → 點擊「+新增產品」→ 產品建立頁（填基本資訊）→ 啟用多規格 → 設定規格層與規格值 → 系統生成 SKU 矩陣 → 逐一填入價格/庫存 → 設定上架時間 → 「儲存並上架」→ 返回產品列表

**Flow 2：CSV 批次匯入產品**
產品列表頁 → 點擊「批次匯入」→ 下載 CSV 模板 → 填寫後上傳 → 查看驗證結果（修正錯誤）→ 確認匯入 → 成功 N 件

**Flow 3：庫存調整與低庫存處理**
產品中心側欄 → 庫存管理 → 切換「⚠ 低庫存」Tab → 找到問題 SKU → Inline 修改庫存 → 填備註 → 確認（自動記錄日誌）

**Flow 4：商家查看並回應貨到通知訂閱**
庫存管理頁 → 「貨到通知」Sub-Tab → 查看訂閱清單 → 補貨後系統自動發信 / 手動標記已通知

## 3. 頁面規格

---

### 3.1 產品列表頁（路徑：/admin/products）

#### 目的
讓商家管理員一覽所有產品，快速篩選/搜尋後執行單筆或批次操作（上下架、複製、刪除、移動分類）。

#### 進入方式
- 點擊左側導覽「產品管理」
- 從產品建立/編輯頁儲存後自動跳回
- 從庫存管理頁麵包屑返回

#### 頁面佈局
- **頂部**：麵包屑（產品中心 > 產品管理）+ 右上角「+ 新增產品」主按鈕（`#303133`，`!rounded-none`）
- **篩選列**：搜尋框 + 分類下拉 + 狀態下拉 + 批次操作 Dropdown（未勾選時 Disabled）
- **主區域**：`<el-table>` 產品列表 + 分頁列

#### 元件清單

| 元件名稱 | 顯示內容 | 互動行為 |
|---------|---------|---------|
| 搜尋框 | Placeholder：「搜尋產品名稱、SKU 編號」 | debounce 300ms 即時過濾；可按 Enter；有清除 × 按鈕 |
| 分類篩選下拉 | 預設「所有分類」；顯示三層縮排分類樹 | 多選；選擇後過濾列表 |
| 狀態篩選下拉 | 全部 / 已上架 / 已下架 / 草稿 / 已排程 | 單選；選擇後過濾列表 |
| 批次操作 Dropdown | 批次上架 / 批次下架 / 移動分類 / 批次刪除 | 未勾選產品時 Disabled；勾選後啟用 |
| 產品列表（`<el-table>`） | 欄位：勾選框、主圖縮圖（48×48）、產品名稱/SKU、分類、狀態 Tag、庫存數、售價、更新時間、操作欄 | 點擊產品名稱→編輯頁；點擊勾選框→批次選取 |
| 狀態 Tag | `<el-tag class="!rounded-full">` 依狀態顯示對應顏色文字 | 僅顯示，不可點擊 |
| 庫存數欄位 | 正常數字；庫存≤閾值時顯示 `⚠ N`（`#F56C6C`） | 僅顯示 |
| 售價欄位 | 有特價時：「特價 $X」(紅) + 「原價 $Y」(刪除線)；0元產品顯示「免費」Tag | 僅顯示 |
| 操作欄 | 「編輯」文字按鈕（藍）、「複製」icon 按鈕（Tooltip: 複製此產品）、「刪除」icon 按鈕（紅） | 編輯→進入編輯頁；複製→新建相同產品草稿；刪除→二次確認 Modal |
| 分頁（`<el-pagination>`）| 每頁 20 筆；顯示總筆數 | 切換頁碼更新列表 |

#### 資料規格

```
Product（產品）:
  - id: BIGINT
  - name: VARCHAR(100)（產品名稱）
  - slug: VARCHAR(100)（前台 URL）
  - status: enum → 見狀態定義
  - price: INT（售價，元）
  - special_price: INT | null（特價）
  - stock: INT（庫存總量，無規格產品用）
  - category_names: string[]（分類名稱清單）
  - main_image_url: string
  - updated_at: datetime
  - is_free: boolean（0元產品）
```

#### 狀態定義

**頁面層級狀態：**

| 狀態 | 觸發條件 | 顯示內容 |
|------|---------|---------|
| 空狀態（Empty） | 尚無任何產品，且無篩選條件 | 插圖（空箱子）+ 「還沒有產品，點擊上方按鈕新增第一件產品吧！」+ 「+ 新增產品」CTA |
| 搜尋無結果 | 有篩選條件但無符合產品 | 「找不到符合的產品」+ 「清除篩選」Button |
| 載入中（Loading） | 資料抓取中 | Skeleton loader（5 列） |
| 錯誤（Error） | 請求失敗 | 「載入失敗，請重試」+ 重試 Button |
| 正常（Default） | 有產品資料 | 產品列表 |

**Status Enum — 產品狀態：**

| 值（英文） | 顯示文字 | 顏色語義 |
|-----------|---------|---------|
| published | 已上架 | Success（綠） |
| unpublished | 已下架 | Info（灰） |
| draft | 草稿 | Warning（黃） |
| scheduled | 已排程 | Primary（藍，`#409EFF`） |

#### 互動行為

| 用戶動作 | 系統反應 |
|---------|---------|
| 點擊「+ 新增產品」 | 跳轉至產品建立頁 |
| 在搜尋欄輸入關鍵字 | debounce 300ms 後即時過濾列表 |
| 點擊分類/狀態篩選 | 列表依條件更新 |
| 勾選產品 → 點擊批次操作 | Dropdown 顯示可執行動作 |
| 點擊「刪除」icon | 彈出確認 Modal（含產品名稱） |
| 點擊「複製」icon | Toast「已複製為草稿」+ 列表頂端出現新草稿產品 |
| 拖曳列表列 | 即時調整 sort_order；Toast「排序已更新」 |

#### 刪除確認 Modal 文案
```
確定要刪除「{產品名稱}」嗎？
此操作無法復原。產品將進入回收狀態，已成立的訂單不受影響。
[取消]  [確定刪除（danger）]
```

批次刪除有進行中訂單時：「{N} 件產品有進行中的訂單，無法刪除，其餘產品已刪除。」

#### 離開方式（Exit Points）
- → 產品建立頁（點擊「+ 新增產品」）
- → 產品編輯頁（點擊產品名稱或「編輯」）
- → 產品列表（刪除/批次操作後留在本頁）

### 3.2 產品建立頁（路徑：/admin/products/create）

#### 目的
讓商家管理員一頁完成產品所有設定，包括基本資訊、圖片、規格 SKU、定價、庫存、SEO、上架排程。

#### 進入方式
- 從產品列表頁點擊「+ 新增產品」

#### 頁面佈局
- **頂部**：麵包屑（產品中心 > 新增產品）+ 右上角操作列（「儲存草稿」次要按鈕 + 「儲存並上架」主按鈕）
- **左主欄（70%）**：基本資訊卡片 → 產品圖片卡片 → 富文本描述卡片 → 規格與定價卡片 → SEO 設定卡片
- **右輔助欄（30%）**：產品狀態卡片 → 上下架檔期卡片 → 產品分類卡片 → 銷售數量顯示卡片 → 進階設定卡片

#### 元件清單

**左主欄元件：**

| 元件名稱 | 顯示內容 | 互動行為 |
|---------|---------|---------|
| 產品名稱輸入框 | Required；Placeholder：「請輸入產品名稱（最多 100 字）」；字元計數 0/100 | 超過 100 字邊框變 `#F56C6C`；錯誤提示顯示在下方 |
| 產品簡短說明 | Textarea；Placeholder：「選填，顯示於產品列表卡片下方（最多 200 字）」；字元計數 | 即時計數 |
| 主圖上傳區 | 正方形拖放區（200×200px 預覽）；Required | 呼叫媒體庫 API；上傳進行中顯示進度條；成功後顯示縮圖 + 重新上傳按鈕 |
| 多圖上傳列 | 最多 9 張附圖（橫向縮圖列表） | 拖曳調整順序；點擊縮圖放大預覽；每張可刪除 |
| 富文本編輯器 | 支援 H2/H3、粗體、斜體、列表、圖片插入（媒體庫）、YouTube 嵌入、表格 | 工具列點擊插入對應格式；圖片插入呼叫媒體庫 |
| SEO 標題 | Placeholder：「留空則自動使用產品名稱」；字元計數 0/70 | 超過 60 字顯示黃色警告：「建議 SEO 標題在 60 字元內」 |
| SEO 描述 | Textarea；Placeholder：「留空則自動截取產品說明前 160 字」；字元計數 | 超過 160 字警告 |
| SEO 關鍵字 | Placeholder：「多個關鍵字以逗號分隔」 | 純文字輸入 |

**右輔助欄元件：**

| 元件名稱 | 顯示內容 | 互動行為 |
|---------|---------|---------|
| 產品狀態選擇 | `<el-select>`：已上架 / 已下架 / 草稿；預設「草稿」 | 選擇「已上架」時，若無上架時間則立即上架 |
| 上架時間選擇器 | `<el-date-picker type="datetime">`；Placeholder：「立即上架（留空）」 | 設定時間後，狀態自動切為「已排程」 |
| 下架時間選擇器 | `<el-date-picker type="datetime">`；Placeholder：「永久上架（留空）」 | 下架時間必須晚於上架時間，否則顯示「下架時間必須晚於上架時間」 |
| 分類選擇器 | `<el-tree-select>` 多選；顯示三層分類樹 | 不選則產品進「未分類」 |
| 銷售數量顯示模式 | `<el-select>`：不顯示 / 顯示實際數字 / 顯示「熱銷中」文字；預設「不顯示」 | 選「顯示實際數字」時出現「起始基數」輸入框 |
| 起始基數 | `<el-input-number>` min=0，預設 0；Tooltip：「前台顯示 = 實際銷售量 + 基數」 | 只在模式為「顯示實際數字」時顯示 |
| 溫層屬性 | `<el-radio-group>`：常溫 / 冷藏（7°C 以下）/ 冷凍（-18°C 以下）；預設「常溫」；附 Tooltip | 選冷藏/冷凍後產品頁顯示溫層 icon（🌡️/❄️）；同時顯示「在產品頁顯示溫層標示」Switch |
| 溫層標示顯示 Switch | `<el-switch>` 預設 ON（冷藏/冷凍時）；標籤：「在產品頁顯示溫層標示」 | 常溫時此 Switch 隱藏 |
| 0元產品 Switch | `<el-switch>` 預設關閉；標籤：「設為免費產品（0 元）」 | 啟用後隱藏所有價格輸入欄，並顯示「每人最大領取數量」設定；啟航方案顯示鎖頭 icon + 「需升級進階電商包」 |
| 每人最大領取數量 | `<el-input-number>` min=1，max=999，預設 1；Tooltip：「限制每個會員最多領取幾件」 | 只在 0元產品啟用時顯示 |
| 0元是否需登入 | `<el-switch>`；Tooltip：「開啟後消費者需登入才能免費領取」 | 只在 0元產品啟用時顯示 |

**規格與定價區塊（單規格預設）：**

| 元件名稱 | 顯示內容 | 互動行為 |
|---------|---------|---------|
| 啟用多規格 Toggle | `<el-switch>` 標籤：「啟用多規格（如顏色、尺寸）」 | 開啟時清空現有單規格數據並彈出確認；展開多規格設定區 |
| 售價 | `<el-input-number>` prefix「$」；Required；min=0 | 0元產品啟用時隱藏 |
| 特價 | `<el-input-number>` prefix「$」；Optional | 特價 ≥ 售價時顯示「特價必須低於售價」 |
| 成本價 | `<el-input-number>` prefix「$」；Optional；Tooltip：「不顯示於前台，僅用於後台毛利計算」 | 純後台欄位 |
| 庫存數量 | `<el-input-number>` min=0；Tooltip：「設為 0 時前台顯示『售完』」 | 輸入 0 時前台不可加入購物車 |
| 低庫存閾值 | `<el-input-number>`；Placeholder：「使用全局設定（N）」 | 留空則用全局設定 |
| SKU 編號 | `<el-input>` 最多 64 字元；Placeholder：「產品貨號（選填）」 | 純文字 |
| 重量 | `<el-input-number>` suffix「公斤」；保留 3 位小數；Tooltip：「用於重量運費計算」 | 留空則不啟用重量運費 |

**規格與定價區塊（多規格模式）：**

| 元件名稱 | 顯示內容 | 互動行為 |
|---------|---------|---------|
| 規格層新增 Button | 「+ 新增規格層」；最多 2 層 | 超過 2 層時 Disabled + Tooltip：「最多設定兩層規格」 |
| 規格層名稱 | `<el-input>` Placeholder：「規格名稱（如：顏色）」；Required；最多 20 字 | 即時驗證重複規格名 |
| 規格值輸入 | Tag 輸入模式；輸入後按 Enter 或逗號確認；每個 Tag 可刪除；最多 10 個 | 超過 10 個顯示「每層最多 10 個規格值」；刪除已有 SKU 的規格值時彈確認 |
| 規格值圖片上傳 | 每個 Tag 右側「上傳圖片」icon；上傳後 Tag 左側顯示 16×16 縮圖 | 呼叫媒體庫 API；Tooltip：「規格代表圖顯示於前台規格選擇器旁」 |
| SKU 矩陣表格 | 自動生成所有規格排列組合；每列可輸入 SKU 編號/售價/特價/庫存/重量；最右欄啟用/停用 Toggle | Tab 鍵在欄位間切換聚焦 |
| 批次填入工具列 | 選擇欄位（售價/特價/庫存）+ 輸入值 + 「套用至所有 SKU」Button | 一鍵批次填入所有 SKU |

#### 資料規格

```
Product（新增產品請求 Body）:
  - name: VARCHAR(100)（Required）
  - short_description: VARCHAR(200)（Optional）
  - description: LONGTEXT（富文本 HTML）
  - main_image_id: BIGINT（媒體庫圖片 ID，Required）
  - image_ids: BIGINT[]（附圖，最多 9 個）
  - category_ids: BIGINT[]（分類 ID）
  - status: enum（published / unpublished / draft / scheduled）
  - publish_at: datetime | null
  - unpublish_at: datetime | null
  - temperature_layer: enum（normal / cold / frozen）
  - show_temp_label: boolean
  - is_free: boolean
  - free_max_per_user: INT（1–999）
  - free_login_required: boolean
  - sales_display_mode: enum（none / count / hot）
  - sales_count_base: INT
  - seo_title: VARCHAR(70)
  - seo_description: VARCHAR(200)
  - seo_keywords: VARCHAR(200)
  - has_variants: boolean（是否多規格）
  - price: INT（單規格時 Required）
  - special_price: INT | null
  - cost_price: INT | null
  - stock: INT
  - low_stock_threshold: INT | null
  - sku_code: VARCHAR(64)
  - weight: DECIMAL(8,3) | null
  - variants: VariantSpec[]（多規格時使用）
  - skus: SKURow[]（多規格時使用）
```

#### 狀態定義

**頁面層級狀態：**

| 狀態 | 觸發條件 | 顯示內容 |
|------|---------|---------|
| 正常（Default） | 進入新增頁 | 空白表單，所有欄位 Placeholder 顯示 |
| 儲存中（Saving） | 點擊儲存按鈕 | 按鈕 Loading 狀態；欄位不可編輯 |
| 儲存成功 | API 回應 201 | Toast「產品已儲存」（草稿）或「產品已上架」（上架）；跳回列表 |
| 儲存失敗 | API 回應 4xx/5xx | Toast（error）「儲存失敗，請重試。若問題持續請聯繫客服。」；停留在頁面 |
| 驗證錯誤 | 必填欄位為空 | 邊框變紅；頁面滾到第一個錯誤欄位 |
| 離開確認 | 有未儲存變更時點擊離開 | 瀏覽器確認對話框：「您有未儲存的變更，確定要離開嗎？」 |

**Status Enum — 溫層：**

## 6. 響應式規格

| 斷點 | 寬度範圍 | 主要版面變化 |
|------|---------|------------|
| Desktop | ≥ 1280px | 完整側欄（240px）+ 左主欄（70%）+ 右輔助欄（30%） |
| Tablet | 768px – 1279px | 側欄收合為圖示（60px）；左右欄維持兩欄 |
| Mobile | < 768px | 側欄隱藏（漢堡選單）；單欄佈局 |

**Mobile 優先頁面（特殊需求）：**

- **產品詳情頁（前台）**：放大鏡改為點擊觸發全螢幕燈箱（Lightbox）
- **多規格 SKU 矩陣**：行動版降格為 Accordion 展示（每個 SKU 為可展開列）
- **後台產品列表**：欄位精簡顯示（隱藏更新時間、成本價等次要欄位）

> 📌 假設：後台整體 Mobile 支援度有限，商家主要在桌面操作；前台消費者頁面（產品頁、貨到通知訂閱）需完整 Mobile 支援。
