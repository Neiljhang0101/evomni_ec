# 前端設計規格缺口清單（PM 決策用）

> **製作視角：** UI/UX 設計師
> **日期：** 2026-05-19
> **用途：** 盤點目前所有 PRD 中，依功能需求應存在前端介面但尚未有完整設計規格的項目，供 PM 決策補寫優先順序及 Prototype 分工。
> **說明：** 本清單不討論後端邏輯，只關注「消費者或商家管理員需要看到什麼、能做什麼操作」的設計空白。已在現有 UX Spec 或 Prototype 中完整規格化的頁面不列入。

---

## 緊急程度說明

| 等級 | 意義 |
|------|------|
| P0 | 缺少此規格會讓工程師無法開工或開工方向錯誤，需在動工前補寫 |
| P1 | 缺少此規格會讓驗收無依據，需在 QA 前補寫 |
| P2 | 建議補寫，可在第一版上線後補充 |

| 影響範圍 | 全站新手引導體驗 |

## 設計師備注：給 PM 的補充說明

1. **前台 Prototype 完全空白是最大的設計風險。** 目前 15 份 Prototype 全為後台，但電商系統的轉換率高度依賴前台消費者體驗，建議優先在 Master PRD §4.3 補入至少以下 4 份前台 Prototype：前台架構（FE-01）、產品詳情頁（FE-03）、購物車（FE-04）、結帳流程（FE-05 + FE-06）。

2. **超商取貨門市選取介面（BE-02）涉及第三方 Widget 整合，** 需要確認綠界 / 藍新提供的地圖 Widget 嵌入方式，再決定是自行設計選取 UI 還是直接跳轉廠商地圖頁，這個決策會影響前台結帳流程的整體設計。

3. **「升級鎖定 UI 統一規範」（BE-04）應優先於各進階功能 Prototype 完成，** 否則各 Prototype 對鎖定狀態的處理方式不一致，上線後視覺體驗雜亂，也可能引發消費者對功能是否可用的誤解。

4. **本清單專注於「缺少設計規格」的項目，** 現有 PRD 差異確認清單（`SPEC/PRD差異確認清單（PM討論用）.md`）另行涵蓋「規格存在但 Prototype 實作有差異」的項目，兩份清單合用才能完整掌握缺口全貌。
