## 全域設定架構

### GS-1. LINE OA 設定 — 層級衝突

| | 說明 |
|---|---|
| **現狀** | 原型將 LINE OA 設定作為全域設定的主選單直屬子項；後台導覽架構規劃與各模組文件（行銷活動規格路徑：「全域設定 > 電商進階設定 > LINE OA 設定」）均認為它是電商進階設定的子項。Prototype 的 PageAdvanced.jsx 已將 `line-oa` 定義為獨立路由，但欄位規格（頻道代碼、頻道金鑰、存取金鑰）與驗證流程在兩份文件中均未深入定義。 |
| **問題** | 層級歸屬不一致，且設定頁面本身的欄位規格尚未補入 PRD。 |
| **來源** | 兩份文件均有 |

**請 PM 確認：** LINE OA 設定要放在全域設定主選單（獨立項目），還是收進電商進階設定之下？確認後，請同步決定 LINE OA 設定頁面的欄位規格（需要哪些憑證、如何驗證串接成功）是否補入全域設定 PRD 或自動旅程 PRD 的前置條件章節。
