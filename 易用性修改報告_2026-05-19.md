### components/pages/PageTracking.jsx

**A05 — 工具白話說明 + ID 取得提示**

ToolCard 元件新增 `desc` prop，在工具名稱下方顯示白話一行描述（僅啟用狀態顯示）。

各工具修改：

| 工具 | 舊標題 | 新標題 | 新描述 |
|---|---|---|---|
| GA4 | Google Analytics 4 | Google 流量分析（GA4） | 追蹤消費者從哪裡來、看了哪些產品、哪個環節離開，幫你了解廣告效益 |
| Meta Pixel | Meta（Facebook）Pixel | Meta（Facebook）廣告像素（Pixel） | 記錄消費者在商店的行為，讓 Facebook / Instagram 廣告投放更精準 |
| CAPI | Meta Conversion API（FB 轉換 API） | Facebook 伺服器轉換（CAPI） | 彌補 iOS 封鎖 Cookie 後遺失的廣告成效數據，從伺服器端直接回傳購買紀錄給 Meta |
| GTM | Google Tag Manager | Google 代碼管理工具（GTM） | 統一管理所有行銷追蹤碼，未來新增第三方工具不需每次請工程師修改程式碼 |

各工具輸入欄位加入 `helperText`，說明到哪裡找對應的 ID：
- GA4：「前往 Google Analytics 後台 > 管理 > 資料串流，點擊串流名稱後即可看到「評估 ID」（G-XXXXXXX）」
- Meta Pixel：「前往 Meta 廣告管理員 > Events Manager（事件管理工具）> 資料來源，可看到 15-16 位的純數字像素編號」
- CAPI：「前往 Meta 廣告管理員 > Events Manager > 設定 > Conversions API，點擊「產生存取權杖」後複製長串英數字」
- GTM：「登入 Google Tag Manager 後台，右上角帳戶名稱旁的 GTM-XXXXXXX 即為容器編號」

欄位標籤同步更新：
- Measurement ID → 評估 ID（Measurement ID）
- Pixel ID → 像素編號（Pixel ID）
- 存取權杖（Access Token）→ 伺服器存取金鑰（Access Token）
- Container ID → 容器編號（Container ID）

TOOL_NAMES 常數全面更新為白話中文（含縮寫）。

- 顯示白話錯誤原因
- 顯示發生時間（小字）
- 加入「前往修復」按鈕，點擊後進入該廠商設定頁面

### components/SharedUI.jsx

**B02 — VendorTypeTag 廠商類型 tooltip**

`VendorTypeTag` 元件加入 hover tooltip，游標移入時顯示各類型的白話說明：

| 類型 | tooltip 說明 |
|---|---|
| 全自動 | 設定完成後，系統自動處理付款確認與出貨通知，幾乎不需人工介入 |
| 設定即用 | 完成基本設定後即可啟用，部分流程需要人工確認 |
| 半人工 | 系統協助部分流程，但出貨或確認仍需手動操作 |
| 人工處理 | 訂單確認及出貨完全需人工操作，適合低訂單量或特殊物流需求 |
