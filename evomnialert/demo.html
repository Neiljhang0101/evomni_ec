EvomniAlert Demo

點擊下方按鈕，實測 4 種訊息類型與進階用法。

基本 4 種 type SiteAlert · 純文字

站台前端靜態內容使用。純文字 API，零 XSS 風險。

SiteAlert.success('送出成功', '我們會盡快聯繫您'); SiteAlert.error('登入失敗', '帳號或密碼錯誤，請重試'); SiteAlert.warning('請確認資料', 'Email 格式不正確'); SiteAlert.info('系統通知', '將於 23:00 進行維護');

支援 HTML 內容 EvomniAlert · DOMPurify

後端動態訊息使用。自動清洗 HTML，僅允許白名單標籤（b, i, strong, em, br, a, span）。

EvomniAlert.show({ type: 'info', title: '系統通知', desc: '請參考 <a href="#" target="_blank">常見問題</a>' });

XSS 防護測試 資安核心

模擬惡意字串注入，驗證 DOMPurify 會清除危險內容。

按下按鈕後，標題中的危險內容會被清除，僅保留安全部分。可打開瀏覽器開發者工具觀察結果。

佇列機制（Queue） 進階

多則訊息同時觸發時會排隊顯示。error 型會插隊到最前方。

// 連續觸發：依序顯示 SiteAlert.success('第 1 則'); SiteAlert.success('第 2 則'); SiteAlert.success('第 3 則'); // error 會插隊到 queue 最前方 SiteAlert.info('一般訊息 A'); SiteAlert.info('一般訊息 B'); SiteAlert.error('錯誤', '會先於 A、B 顯示');

fromResponse（AJAX 快捷） 後端整合

模擬 AJAX 回應，自動依 ok 與 type 欄位顯示對應訊息。

EvomniAlert.fromResponse({ ok: true, title: '送出成功', msg: '我們會盡快聯繫您' });

logout 快捷 登出流程

Plan B 登出專用：成功圖示 + OK 鈕 + 按 OK 後導向指定路徑。不自動關、不可 backdrop 關。

// 最簡：按 OK 後 location.href = '/' SiteAlert.logout(); // 指定跳轉路徑 SiteAlert.logout({ redirect: '/zh-tw/member/login' }); // 先清 token 再跳轉 SiteAlert.logout({ onConfirm: () => { localStorage.clear(); location.href = '/'; } });
Demo 中用 onConfirm 攔截導向，避免按 OK 真的跳頁離開 demo。

dismiss（主動關閉） 進階

取得 show() 回傳的 id，可在非同步完成時主動關閉。

const id = EvomniAlert.show({ type: 'info', title: '載入中...', duration: 0 // 不自動關閉 }); setTimeout(() => { EvomniAlert.dismiss(id); }, 1500);
