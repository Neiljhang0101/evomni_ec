# PRD｜系統分頁管理模組

**文件版本：** v1.0  
**建立日期：** 2026-05-28  
**狀態：** 草稿

---

## 1. 背景與目標

### 1.1 背景

電商網站除了主選單可見的功能頁外，存在一類**必要但不對外顯示於選單**的頁面（如隱私權政策、404 錯誤頁等）。這類頁面過去散落在各功能模組中，缺乏統一管理介面，造成：

- 編輯入口難以找到
- 上線前容易遺漏
- 無法統一確認各頁面狀態（已設定 / 未設定 / 停用）

### 1.2 目標

建立「**系統分頁**」管理區塊，提供後台管理者統一檢視、編輯與維護所有系統必要頁面的介面。

### 1.3 範疇

| 範疇 | 說明 |
|------|------|
| **包含** | 法律合規頁、功能導向頁、錯誤頁、行銷轉換頁 |
| **不包含** | 主選單頁面、商品頁、部落格文章 |

## 2. 使用者角色

| 角色 | 說明 | 操作權限 |
|------|------|---------|
| 超級管理員 | 平台擁有者 | 全部讀寫 |
| 一般管理員 | 網站管理人員 | 可編輯內容，不能修改網址 |

---

## 3. 系統分頁清單定義

### 3.1 法律合規類

| 頁面識別碼 | 頁面名稱 | 預設路徑 | 必填 | 說明 |
|-----------|---------|---------|------|------|
| `privacy-policy` | 隱私權政策 | `/privacy-policy` | ✅ | 說明個人資料蒐集方式與用途 |
| `terms-of-service` | 服務條款 | `/terms-of-service` | ✅ | 使用者與平台之間的使用協議 |
| `cookie-policy` | Cookie 政策 | `/cookie-policy` | ⚠️ 建議 | 若有使用追蹤 Cookie 則必填 |
| `disclaimer` | 免責聲明 | `/disclaimer` | ⚠️ 建議 | 依產業合規需求 |
| `refund-policy` | 退款政策 | `/refund-policy` | ✅ 電商必填 | 退款條件與流程說明 |
| `return-policy` | 退換貨政策 | `/return-policy` | ✅ 電商必填 | 退換貨規則與期限 |
| `copyright` | 版權聲明 | `/copyright` | ⚠️ 建議 | 網站內容著作權說明 |

### 3.2 功能性頁面

| 頁面識別碼 | 頁面名稱 | 預設路徑 | 必填 | 說明 |
|-----------|---------|---------|------|------|
| `404` | 找不到頁面 | 系統自動觸發 | ✅ | 使用者訪問不存在路徑時顯示 |
| `500` | 伺服器錯誤 | 系統自動觸發 | ✅ | 系統異常時顯示 |
| `maintenance` | 維護中頁面 | 系統自動觸發 | ✅ | 系統維護期間全站覆蓋顯示 |

## 4. 功能需求

### 4.1 系統分頁管理列表

**入口位置：** 後台 > 網站設計 > 系統分頁

#### 列表欄位

| 欄位 | 說明 |
|------|------|
| 頁面名稱 | 顯示名稱（可多語系） |
| 狀態 | 啟用 / 停用 |
| 最後更新 | 最後一次儲存的時間與操作者 |
| 操作 | 編輯、預覽 |

#### 狀態定義

| 狀態 | 說明 | 視覺標示 |
|------|------|---------|
| 啟用 | 啟用中 | 綠色標籤 |
| 停用 | 暫時關閉 | 灰色標籤 |

#### 篩選與搜尋

- 依**狀態**篩選
- 關鍵字搜尋頁面名稱

#### 完成度儀表板（頁面頂部）

以進度條或數字摘要顯示整體設定完成率：

```
系統分頁完成度：14 / 19   ████████████░░░  73%
⚠️ 5 個必填頁面尚未設定
```

---

### 4.2 頁面編輯

每個系統分頁點擊「編輯」後進入獨立的編輯頁，包含：

#### 基本設定

| 欄位 | 類型 | 說明 |
|------|------|------|
| 頁面標題 | 文字輸入 | 顯示於瀏覽器 Tab 與 SEO |
| 路徑（Slug） | 文字輸入 | URL 路徑，部分頁面固定不可改 |
| 狀態 | 開關 | 啟用 / 停用 |
| 頁面內容 | 文字輸入（160字） | 頁面內容 |
| SEO 描述 | 文字輸入（160字） | SEO 描述 |

#### 內容編輯

- 使用**富文字編輯器**（WYSIWYG）
- 支援插入圖片、表格、連結
- 錯誤頁（404、500、維護中）額外提供：
  - 自訂標題文字
  - 自訂說明文字
  - 自訂按鈕文字與導向連結
  - 自訂背景圖或插圖

#### 多語系

若網站啟用多語系，編輯器頂部出現語言切換 Tab（繁中 / EN / ...），各語系獨立填寫。

#### 儲存行為

- 自動儲存草稿（每 30 秒）
- 手動「儲存並發布」
- 儲存後列表狀態自動更新為「已設定」

---


### 4.4 頁面預覽

- 點擊「預覽」可在新分頁以實際前台樣式預覽該系統分頁
- 預覽時頂部顯示提示條：「目前為預覽模式，此頁面對訪客不可見」

## 5. 非功能性需求

| 項目 | 需求 |
|------|------|
| 權限控制 | 各角色權限依 §2 定義執行 |
| 多語系 | 配合網站語系設定顯示對應 Tab |
| SEO | 所有系統分頁支援獨立 Meta Title / Description |
| 響應式 | 後台管理介面支援桌機，前台頁面支援手機 |
| 效能 | 404 / 500 / 維護頁靜態化，不依賴資料庫查詢 |
| 日誌 | 維護模式的啟用 / 關閉記錄操作者與時間 |

## 7. 開發任務拆解

### 後端

- [ ] `SystemPage` 資料模型建立（id、slug、type、status、content、meta、updated_at、updated_by）
- [ ] CRUD API：`GET /api/system-pages`、`GET /api/system-pages/:id`、`PUT /api/system-pages/:id`
- [ ] 維護模式 API：`POST /api/maintenance/toggle`
- [ ] IP 白名單 middleware
- [ ] 靜態頁面快取（404、500、maintenance）

### 前端後台

- [ ] 系統分頁列表頁（含篩選、完成度儀表板）
- [ ] 系統分頁編輯頁（富文字 + SEO 欄位 + 多語系 Tab）
- [ ] 維護模式專屬設定區塊
- [ ] 後台全域維護模式警示橫幅

### 前台

- [ ] 各系統分頁渲染模板
- [ ] 維護模式全站攔截 middleware
- [ ] 404 / 500 靜態模板

## 8. 驗收標準

| 項目 | 驗收條件 |
|------|---------|
| 列表顯示 | 所有系統分頁正確列出，狀態與實際內容同步 |
| 完成度計算 | 必填頁面數量正確，未設定數量正確 |
| 內容編輯 | 儲存後前台正確顯示，狀態更新為「已設定」 |
| 維護模式 | 開啟後前台訪客只看到維護頁，白名單 IP 正常瀏覽，後台顯示警示 |
| 404 頁面 | 輸入不存在路徑正確顯示自訂 404 內容 |
| 多語系 | 切換語系後顯示對應語言內容 |
| 權限 | 編輯人員無法變更頁面狀態 |
