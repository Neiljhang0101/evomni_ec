## 2. 產品背景與目標

### 2.1 背景

Evomni 後台「全域設定」選單為 CMS 原有功能，電商模組啟用後，於現有選單下方新增電商專屬設定項目，不改動 CMS 原有選單結構。

目前 CMS 系統設定 > 基本設定已涵蓋：商店名稱、Logo、地址、電話、傳真、Email。**電商基本設定不重複這些欄位**，僅補充 CMS 沒有但電商運作必需的設定。

### 2.2 設計原則

- **不重複**：CMS 已有的設定沿用，電商設定頁僅補充「純電商才需要」的欄位
- **單一來源**：商家修改商店名稱、地址只需在一處操作，電商全站自動引用 CMS 系統設定
- **分層清晰**：電商基本設定（商業資訊）vs. 電商進階設定（系統行為參數）

## 5. 功能需求

### 5.1 功能範圍

| 頁面 | 路徑 | 頁面類型 | 說明 |
| --- | --- | --- | --- |
| 電商基本設定 | 全域設定 > 電商基本設定 | SETTINGS | 幣別、訂單編號格式、電子發票統一編號 |
| 電商進階設定 | 全域設定 > 電商進階設定 | SETTINGS | 購物車逾時、庫存警示門檻、及其他功能設定入口 |

### 5.2 電商基本設定 — 欄位邏輯

#### 幣別
- 商家選擇主要交易幣別，影響所有訂單金額顯示、發票金額單位、數據中心報表合計基準
- **v1.0：單一幣別**，商家選一種幣別套用全站；首次儲存後即鎖定，不可修改（防止已出訂單金額與新訂單幣別不一致）
- **Phase 2 多幣別**：支援商家同時設定多個可接受幣別、匯率基準、各幣別獨立報表合計；DB 設計請預留此擴充空間（見 §7.1）
- 預設值：TWD（新台幣）
- v1.0 支援幣別選項：TWD / USD / HKD / JPY / SGD / EUR / GBP（7 種，涵蓋多語系商家主要市場）

#### 訂單編號格式
- 由「前綴字元」+「序號長度」組成，系統自動拼接，每筆訂單序號自動遞增
- 前綴範例：`ORD`、`ORDER`、商家英文縮寫（最多 10 字元，限英文大寫字母與數字）
- 序號長度：可選 4、5、6 位（不足補 0，例：00001）
- 即時預覽拼接結果（見 §6 UI 規格）
- **首次儲存後即鎖定（唯讀）**，不以「有無訂單」為條件——設計意圖是「開站前設好、設定後不改」（同 Shopify / SHOPLINE 做法）；需更改者請聯繫營運輔導顧問

#### 發票資訊（電子發票商家抬頭）
以下兩欄位用於電子發票的「賣方資訊」欄位，與 CMS 系統設定的品牌名稱獨立維護（業界標準做法：Shopify 分 Store name / Legal name of business；SHOPLINE、91App 亦同）。

- **發票公司抬頭**：法人登記名稱（必填），印在電子發票賣方欄，必須與商業登記一致。例：「威德科技有限公司」
- **統一編號**：8 碼數字（選填）；有填時啟用 B2B 開立功能，消費者結帳可選擇「公司戶」輸入買方統編
- 此區塊為「商家賣方資訊」；電子發票 API 串接金鑰（憑證/環境設定）在「金物流串接 > 電子發票設定」中設定

### 5.3 電商進階設定 — 欄位邏輯

#### 購物車逾時
- 消費者將商品加入購物車後，若超過設定分鐘數未完成結帳，購物車自動清空
- 清空時可觸發「購物車挽回」行銷旅程（若已啟用）
- 預設值：60 分鐘；可設範圍：15 分鐘 ～ 1440 分鐘（24 小時）
- 設定值同時作為行銷旅程「購物車閒置 N 分鐘觸發條件」的依據

#### 庫存警示門檻
- 全局低庫存閾值，當商品 / SKU 庫存數量 ≤ 此值，系統每日發送警示通知給商家管理員
- 預設值：5；可設範圍：1 ～ 999
- 各商品 / SKU 可在「商品中心 > 商品設定」中覆寫個別閾值；若個別未設定，套用此全局值
- 同一 SKU 7 天內只發一次通知（防止重複轟炸）
- 通知方式：Email（發送至 CMS 系統設定 > 基本設定的商家 Email）

#### 其他設定（入口連結，各有獨立 PRD）
- **第三方登入設定**：LINE / Google / Facebook OAuth 串接金鑰設定 → 詳見 `Evomni_第三方登入_PRD.md`
- **LINE OA 設定**：LINE 官方帳號串接（Channel ID / Channel Secret / Access Token）→ 詳見 Part4 §6.4
- **會員驗證設定**：消費者前台申請會員的驗證方式（Email 連結 / 簡訊 OTP）→ 詳見 Part6 §6.10

#### C. 互動細節

- **幣別與訂單前綴：首次儲存後鎖定**：後端記錄 `settings_locked = true`；頁面載入時若已鎖定，幣別與前綴/序號長度欄位一律 `disabled` 顯示為灰色唯讀文字，並顯示 ⓘ 說明（R7：不說「DB flag」，用「設定後不可修改」）
- **幣別首次儲存前警示**：儲存前顯示 `<el-message-box confirm>`：「幣別儲存後將無法修改，請確認選擇的幣別正確。」確認後才寫入
- **訂單編號即時預覽**：前綴或序號長度任一變動後立即更新預覽字串（前端純計算，無 API call）
- **儲存成功**：`<el-message type="success">` 顯示「設定已儲存」（2 秒後消失）
- **未儲存離開防呆**：若有未儲存變更，離開頁面前顯示 `<el-message-box confirm>`「尚有未儲存的設定，確定離開？」

## 7. 資料與技術規格（建議參考）

### 7.1 DB Schema 建議

```sql
-- system_settings 表（新增電商設定欄位）
currency                    VARCHAR(3)   DEFAULT 'TWD',     -- 幣別代碼 ISO 4217；v1.0 單一幣別
                                                            -- Phase 2 多幣別：建議改為獨立 shop_currencies
                                                            -- 表（shop_id, currency_code, is_primary, exchange_rate）
ecommerce_settings_locked   BOOLEAN      DEFAULT FALSE,     -- 首次儲存後設為 TRUE，鎖定幣別與訂單前綴
order_number_prefix         VARCHAR(10)  DEFAULT 'ORD',     -- 訂單編號前綴（鎖定後不可更新）
order_number_seq_length     TINYINT      DEFAULT 5,         -- 序號位數 4/5/6（鎖定後不可更新）
order_number_seq_current    INT          DEFAULT 0,         -- 當前最大序號（每張訂單遞增）
invoice_company_name        VARCHAR(60)  DEFAULT NULL,      -- 發票公司抬頭（法人名稱）
tax_id                      CHAR(8)      DEFAULT NULL,      -- 電子發票統一編號（賣方）
```
